微服务开发技术篇-微服务网关

 2018-08-28   作者: TanDi

为什么需要网关

没有用网关之前

  • 客户端需要维护众多服务的地址
  • 每个服务都需要对客户端请求做权限验证
  • 跨域请求处理变得复杂
  • 拦截请求责任落在服务上领到服务变得沉重
  • 对用户数据采集不方便

引入服务网关后

  • 易于监控
  • 易于认证
  • 解决了没有网关前的各种问题，网关就像面向客户端的Facede模式

微服务网关和Nginx有什么区别？

  首先要说的是Nginx也可以作为微服务网关来使用，而且性能也非常不错，但需要借助Lua脚本来手动实现各种功能需求，因此增加了学习和开发的成本(当然，如果确实不嫌麻烦和喜欢定制的话其实也可以使用Node.js来手动实现一个微服务网关)。随着微服务的兴起其实诞生了不少既简单又好用的微服务网关，目前也有不少企业把Nginx和微服务网关结合来使用。

常用微服务网关

  • Netflix Zuul 1.x：SpringCloud集成，已停止更新
  • Netflix Zuul 2.x ：无SpringCloud集成
  • Mashape Kong ：面向Cloud Native，基于Nginx_Lua模块开发
  • SpringCloud Gateway ：Zuul的代替品， 基于Sprint Boot2、WebFlux、Project Reactor构建
  • Linkerd ：纳入了CNCF，即面向Cloud Native

目前而言个人较为推荐 Kong 和 SpringCloud Gateway。

基础概念和处理流程

  • Route：是 ID、URI、Predicates、Filters 的集合
  • Predicate：根据配置来匹配HTTP请求，HTTP请求必须满足Predicate要求
  • Filter：过滤请求，可以在发送请求之前或之后修改请求和响应

RequestRateLimiter（目前需要借助Redis实现）

详细参考： https://cloud.spring.io/spring-cloud-gateway/reference/html/#requestratelimiter-gatewayfilter-factory

  • Gateway限流需借助 KeyResolver 和 RateLimiter 实现
  • KeyResolver：可插拔策略，用于设置限流关键字（key），例如ip
  • RateLimiter ：限流算法实现
  • 目前两个接口只分别提供了 PrincipalNameKeyResolver、RedisRateLimiter 实现。官方文档中表示，在未来版本将有不同的KeyResolver和RateLimiter实现

如果自定义，配置方法如下

spring:
  cloud:
    gateway:
      routes:
        - id: test
          uri: http://localhost:8080
          filters:
            - name: RequestRateLimiter
              args:
                  KeyResolver参数名称：#{@myKeyResolver} # KeyResolver的Bean名
                RedisRateLimiter参数名称: 值

【例子】 - RedisRateLimiter 令牌桶算法限流例子。需要添加 spring-boot-starter-data-redis-reactive 依赖和Redis支持

spring:
  cloud:
    gateway:
      routes:
        - id: test
          uri: http://localhost:8080
          filters:
            - name: RequestRateLimiter
              args:
                  # 解析器的 Bean 对象的名字
                  key-resolver: '#{@pathKeyResolver}'
                  # 令牌桶每秒填充令牌数（请求速率）
                redis-rate-limiter.replenishRate: 10
                # 令牌桶容量数，即1秒钟允许的最大用户请求数
                redis-rate-limiter.burstCapacity: 20
redis:
    host: 127.0.0.1
    port: 6379

依赖

<dependency>
    <groupId>org.springframework.boot</groupId>

代码

@Configuration
public class KeyResolverConfiguration {
       // 按照用户限流
    @Bean
    KeyResolver userKeyResolver() {
        return exchange -> Mono.just(exchange.getRequest().getQueryParams().getFirst("user"));

}

Retry

失败重试

spring:
  cloud:
    gateway:
      routes:
        - id: test
          uri: http://localhost:8080
          predicates:
            - Path=/
          filters:
          - name: Retry
            args:
              retries: 3 # 重试次数
              statuses: BAD_GATEWAY # 应重试的HTTP状态代码
              backoff: # 补偿指数（一般补偿指的是重新操作）
                firstBackoff: 10ms
                maxBackoff: 50ms # 最大重试时间
                factor: 2 # 迭代因子
                # 第一次重试在：firstBackoff * (factor ^ n) 之后执行，其中n为迭代次数retries
                # 即失败后80ms后进行重试，重试时间限制在maxBackoff

ModifyRequestBody

修改请求体

  • 文档中提到该拦截器是bate版，后续版本可能会修改
  • ModifyRequestBody拦截器不支持配置文件配置，只能通过Java配置
@SpringBootApplication
public class MsaGatewayApplication {
    public static void main(String[] args) {
        SpringApplication.run(MsaGatewayApplication.class, args);
    }

    @Bean
    public RouteLocator routes(RouteLocatorBuilder builder) throws Exception {
        return builder.routes().route("modify_request_body", r -> {
            // 拦截客户端 localhost:3333 请求
            return r.path("/").filters(f -> {
                // String.class：输出类型
                // Hello.class：输出类型
                // MediaType.APPLICATION_JSON_VALUE：修改成新的Content-Type类型
                return f.modifyRequestBody(String.class, Hello.class, MediaType.APPLICATION_JSON_VALUE, (exchange, s) -> {
                    return Mono.just(new Hello(s.toUpperCase()));
                });
            }).uri("http://localhost:8080");
        }).build();
    }

    static class Hello {
        String message;
        public Hello() { }
        public Hello(String message) {
            this.message = message;
        }
        public String getMessage() {
            return message;
        }
        public void setMessage(String message) {
            this.message = message;
        }
    }
}

ModifyResponseBody

修改响应体

@SpringBootApplication
public class MsaGatewayApplication {
    public static void main(String[] args) {
        SpringApplication.run(MsaGatewayApplication.class, args);
    }

    @Autowired
    private ObjectMapper objectMapper;

    @Bean
    public RouteLocator routes(RouteLocatorBuilder builder) throws Exception {
        return builder.routes().route("modify_response_body", r -> {
            // 拦截客户端 localhost:3333 请求
            return r.path("/").filters(f -> {
                // String.class：输出类型
                // Hello.class：输出类型
                // MediaType.APPLICATION_JSON_VALUE：修改成新的Content-Type类型
                return f.modifyResponseBody(String.class, String.class, MediaType.APPLICATION_JSON_VALUE, (exchange, s) -> {
                    Hello h = new Hello(s.toUpperCase(), 200);
                    String json = null;
                    try {
                        json = objectMapper.writeValueAsString(h);
                    } catch (JsonProcessingException e) {
                        e.printStackTrace();
                    }
                    return Mono.just(json);
                });
            }).uri("http://localhost:8080");
        }).build();
    }

    static class Hello {
        String message;
        int status;
        public Hello() { }
        public Hello(String message, int status) {
            this.message = message;
            this.status = status;
        }
        public int getStatus() {
            return status;
        }

        public void setStatus(int status) {
            this.status = status;
        }

        public String getMessage() {
            return message;
        }
        public void setMessage(String message) {
            this.message = message;
        }
    }
}

Java API Routers (编程式配置)例子

@Bean
public RouteLocator customRouteLocator(RouteLocatorBuilder builder, ThrottleGatewayFilterFactory throttle) {
    return builder.routes()
            .route(r -> r.host("**.abc.org").and().path("/image/png")
                .filters(f ->
                        f.addResponseHeader("X-TestHeader", "foobar"))
                .uri("http://httpbin.org:80")
            )
            .route(r -> r.path("/image/webp")
                .filters(f ->
                        f.addResponseHeader("X-AnotherHeader", "baz"))
                .uri("http://httpbin.org:80")
                .metadata("key", "value")
            )
            .route(r -> r.order(-1)
                .host("**.throttle.org").and().path("/get")
                .filters(f -> f.filter(throttle.apply(1,
                        1,
                        10,
                        TimeUnit.SECONDS)))
                .uri("http://httpbin.org:80")
                .metadata("key", "value")
            )
            .build();
}

开启Hystrix和注册中心（Ribbon）

@RestController
@EnableEurekaClient
@SpringBootApplication
public class MsaGatewayApplication {
    public static void main(String[] args) {
        SpringApplication.run(MsaGatewayApplication.class, args);
    }

    @Bean
    public RouteLocator routeLocator(RouteLocatorBuilder builder) {
        // lb： 表示从注册中心获取服务
        return builder.routes()
                .route(r ->
                        r.path("/gateway/**").filters(
                                // stripPrefix(1) 会自动去掉 /gateway
                                f -> f.stripPrefix(1).hystrix(config ->
                                        config.setName("gateway")
                                                .setFallbackUri("forward:/fallback")))
                                .uri("lb://msa-eureka-provider-user"))
                .build();
    }

    @RequestMapping("/fallback")
    public Mono<String> fallback() {
        return Mono.just("服务繁忙请稍后重试！");
    }
}
