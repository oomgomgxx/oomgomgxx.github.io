Docker的基本使用

 2018-07-23   作者: TanDi

认识Docker

  • Docker是一个应用容器引擎，可实现环境的一次封装到处运行
  • Docker支持多容器同时运行，利用容器可以实现产品的：标准化打包、环境隔离、标准化部署等功能

个人感受：

  作为一个开发者来说可能Docker并不是你百分需要掌握的技能，但是如果你会Docker的话，个人认为这将会大大提高你的学习和生产能力。

  就我个人而言就最为讨厌配置环境，因为环境不同或者稍有些许配置错误都将会浪费很多时间，这使得我不能专注于技术本身而将时间浪费在了弄环境上。这时Docker（或者说容器技术）就打救了我。使用Docker只需一两条命令就很便捷地搭建出一个数据库或各种应用服务，设置各种集群等，无疑这对于广大开发者来说是一种福音。

容器化与虚拟化的区别

  • 容器化可以简单理解为现今的一种轻量级虚拟化技术
  • 容器引擎主要依赖操作系统的隔离技术来实现容器的虚拟和隔离。例如Linux的network namespace可以做到网卡隔离。而cgroup则可以做到进程隔离等
  • 虚拟化是从物理硬件上虚拟出一套虚拟硬件，所以位于虚拟化上的应用限制于虚拟硬件（Hypervisor层)。而容器技术则直接受限于物理硬件，换句话就是容器技术可以更加充分地利用物理机上的资源。这点是作为使用者应该知道的最重要的区别。

组件介绍

  • Docker Client：负责接收用户的 Docker Command 和 Restful API
  • Docker Deamon：可理解为Docker Server，即我们所理解的容器引擎。负责处理Docker Client请求并协调处理其他Docker组件
  • Docker Image：镜像可以理解为对某个应用及其环境的封装。在Docker中镜像的特点是：只读、可重用和可叠加
  • Docker Container：容器是镜像的运行时。在Docker中容器之间可以做到资源隔离，这就是所谓的” 船舱隔离 “
  • Docker Regitry：Regitry是Docker Image的托管仓库。Regitry可以分为公共和私有两种。公司内部或者个人可以搭建自己的私有仓库在保存一些不想被外界使用的镜像。而一般的开源应用服务如果支持容器化的话都会将自己的镜像放到公共仓库中提供给大家使用（类似于maven中央仓库概念）。例如我们现在想要安装一个MySQL，这时我们就可以到公共仓库中找到对应的镜像文件然后拉下来使用即可。

docker run常用参数

-a stdin: 指定标准输入输出内容类型，可选 STDIN/STDOUT/STDERR 三项；

-d: 后台运行容器，并返回容器ID；

-i: 以交互模式运行容器，通常与 -t 同时使用；

-p: 端口映射，格式为：主机(宿主)端口:容器端口；

-t: 为容器重新分配一个伪输入终端；

--name="nginx-lb": 为容器指定一个名称；

--dns 8.8.8.8: 指定容器使用的DNS服务器，默认和宿主一致；

--dns-search example.com: 指定容器DNS搜索域名，默认和宿主一致；

-h "mars": 指定容器的hostname；

-e username="ritchie": 设置环境变量；

--env-file=[]: 从指定文件读入环境变量；

--cpuset="0-2" or --cpuset="0,1,2": 绑定容器到指定CPU运行；

-m :设置容器使用内存最大值；

--net="bridge": 指定容器的网络模式，支持 bridge/host/none/container: 四种类型；

--link=[]: 添加链接(连接的容器ip会配合到DNS中)；

--expose=[]: 开放一个端口或一组端口；

--restart=no ：指定宕机重启策略，支持no/no-failure/always/unless-stopped

构建镜像

了解镜像的构建

  • 首先 Docker Image 本身是只读且可复用的。那么如何复用呢？在Docker中镜像复用是通过镜像叠加来实现的
  • 如上图所示，因为镜像本身是只读的，所以每一次修改都会生成一个新的镜像层（layer）
  • 图中的 bootfs（引导文件系统） 主要用于加载和初始化内核相关内容，完成后会自动unmount。而 Base Image 是整个镜像的最基层，即 rootfs（根文件系统），可以简单理解为是一个最简约轻量的文件系统。随后的 emacs、apache 等等实质都是一些软件应用的打包，而在容器运行时，容器将会包含所有镜像中的内容

Dockerfile语法

FROM
# 基础镜像（注意：FROM必须在文件顶部）
FROM 镜像名称

  • scratch 是基础镜像的特殊值，表示当前镜像不基于任何镜像来构建镜像，例如只用作软件打包

  • 最基础的镜像应该是系统镜像

  • 如果是基于某些软件应用来构建自己的镜像，则无需显式指定系统镜像而只需直接基于某个软件应用镜像来构建即可。例如 FROM nginx，这里的nginx本身已经是基于系统镜像来构建的，所有已经有了 rootfs。因此在构建自己的镜像时，我们只需考虑如何使用基础镜像和挂载数据到基础镜像即可。如下：

RUN
# 镜像构建时执行的命令(注意：每执行一次RUN都会生成新的Layer)
RUN 需要执行的linux命令
MAINTAINER
# 指定Dockerfile的元数据。例如创建者的信息
MAINTAINER 作者名称<作者邮箱>
ADD
# 添加文件到镜像的指定位置(因为镜像是只读的，所以该操作会产生新Layer，其他指令同理)
# 源文件可以是一个路径也可以是一个url地址（注意：会自动进行解压）
ADD 源文件 目标文件
ADD ["源文件1", "源文件2", ..., "目标文件"]
COPY
# 添加文件到镜像的指定位置
# 功能和ADD一样，但不支持解压和URL
COPY 源文件 目标文件
COPY ["源文件1", "源文件2", ..., "目标文件"]
ENV
# 设置环境变量
ENV key value
ENV key=value
ARG
# 指定构建时参数，功能和ENV一样，但是只在Dockerfile内有效
ARG key=value

LABEL

# 指定镜像元数据。可以使用 docker inspect 命令查看到
LABEL 元数据
LABEL version="2.0"
LABEL desc="其实就是描述信息"
WORKDIR
# 指定工作目录
# 注意：该指令相当于Linux的CD命令，在该命令之后的RUN/CMD/ENTRYPOINT，都将在该工作目录下进行
WORKDIR 目录路径
VOLUME
# 指定数据挂载点。可将容器数据目录挂载到宿主机器上
# 非常重要，用于避免数据因容器重启而丢失
# 可以使用 docker volume ls 命令查看到
# 启动容器 docker run 的时候，可以通过-v参数修改挂载点
VOLUME 路径
EXPOSE
# 暴露容器端口（只针对容器之间）
# 注意1：该配置只作声明作用，有助于观看Dockerfile的人得知端口信息
# 注意2：如果外部需要访问，则使用参数-p
EXPOSE 端口号
EXPOSE 端口号/协议
CMD
# 设置容器启动时执行的默认命令
# 注意1: CMD在Dockerfile中是唯一的，既即使定义多个CMD都会被最后的覆盖
CMD ["命令"]
CMD ["命令","命令参数1","命令参数2"]
CMD 命令 命令参数
# 注意2：CMD只是默认的，如果docker run的指定了命令则CMD会被覆盖
# 例如：docker run -it tomcat bin/bash
ENTRYPOINT
# 功能和CMD一样，但不会被 docker run 的命令行参数指定的指令所覆盖
# 如：ENTRYPOINT ["/bin/echo"]，然后docker run -itd xxxx hi传入参数，则会输出hi
ENTRYPOINT ["命令"]
ENTRYPOINT ["命令","命令参数1","命令参数2"]
ENTRYPOINT 命令 命令参数
RUN、CMD、ENTRYPOINT之间的区别

说明：

  1. 如果ENTRYPOINT在CMD之后定义，ENTRYPOINT中的命令会覆盖CMD中的命令
  2. 而且ENTRYPOINT中的命令，可以使用CMD中定义的参数
  3. ENTRYPOINT使用参数的顺序：先使用自己定义的参数，再使用CMD中的定义的参数

  • 注意：shell格式能输出ENV变量，而exec格式不能

FROM debian:stretch-slim

# 在debian layer(临时产生的容器)中添加用户和用户组
# 该操作会生成新的layer
RUN groupadd -r mysql && useradd -r -g mysql mysql

# 更新上一层layer(临时产生的容器)和下载配置相关上下文
# 该操作会生成新的layer
RUN apt-get update && apt-get install -y --no-install-recommends gnupg dirmngr && rm -rf /var/lib/apt/lists/*

# 设置环境变量
ENV GOSU_VERSION 1.7

# 更新上一层layer(临时产生的容器)和配置相关上下文
# 该操作会生成新的layer
RUN set -x \
    && apt-get update && apt-get install -y --no-install-recommends ca-certificates wget && rm -rf /var/lib/apt/lists/* \
    && wget -O /usr/local/bin/gosu "https://github.com/tianon/gosu/releases/download/$GOSU_VERSION/gosu-$(dpkg --print-architecture)" \
    && wget -O /usr/local/bin/gosu.asc "https://github.com/tianon/gosu/releases/download/$GOSU_VERSION/gosu-$(dpkg --print-architecture).asc" \
    && export GNUPGHOME="$(mktemp -d)" \
    && gpg --batch --keyserver ha.pool.sks-keyservers.net --recv-keys B42F6819007F00F88E364FD4036A9C25BF357DD4 \
    && gpg --batch --verify /usr/local/bin/gosu.asc /usr/local/bin/gosu \
    && gpgconf --kill all \
    && rm -rf "$GNUPGHOME" /usr/local/bin/gosu.asc \
    && chmod +x /usr/local/bin/gosu \
    && gosu nobody true \
    && apt-get purge -y --auto-remove ca-certificates wget

# 创建文件夹docker-entrypoint-initdb.d
# 该操作会生成新的layer
RUN mkdir /docker-entrypoint-initdb.d

# 更新上一层layer(临时产生的容器)和配置相关上下文
# 该操作会生成新的layer
RUN apt-get update && apt-get install -y --no-install-recommends \
# for MYSQL_RANDOM_ROOT_PASSWORD
        pwgen \
        openssl \
        perl \
    && rm -rf /var/lib/apt/lists/*

# 配置相关上下文
# 该操作会生成新的layer
RUN set -ex; \
# gpg: key 5072E1F5: public key "MySQL Release Engineering <mysql-build@oss.oracle.com>" imported
    key='A4A9406876FCBD3C456770C88C718D3B5072E1F5'; \
    export GNUPGHOME="$(mktemp -d)"; \
    gpg --batch --keyserver ha.pool.sks-keyservers.net --recv-keys "$key"; \
    gpg --batch --export "$key" > /etc/apt/trusted.gpg.d/mysql.gpg; \
    gpgconf --kill all; \
    rm -rf "$GNUPGHOME"; \
    apt-key list > /dev/null

# 设置环境变量
ENV MYSQL_MAJOR 8.0
ENV MYSQL_VERSION 8.0.18-1debian9

# 配置相关上下文
# 该操作会生成新的layer
RUN echo "deb http://repo.mysql.com/apt/debian/ stretch mysql-${MYSQL_MAJOR}" > /etc/apt/sources.list.d/mysql.list

# 更新上一层layer(临时产生的容器)和配置相关上下文
# 该操作会生成新的layer
RUN { \
        echo mysql-community-server mysql-community-server/data-dir select ''; \
        echo mysql-community-server mysql-community-server/root-pass password ''; \
        echo mysql-community-server mysql-community-server/re-root-pass password ''; \
        echo mysql-community-server mysql-community-server/remove-test-db select false; \
    } | debconf-set-selections \
    && apt-get update && apt-get install -y mysql-community-client="${MYSQL_VERSION}" mysql-community-server-core="${MYSQL_VERSION}" && rm -rf /var/lib/apt/lists/* \
    && rm -rf /var/lib/mysql && mkdir -p /var/lib/mysql /var/run/mysqld \
    && chown -R mysql:mysql /var/lib/mysql /var/run/mysqld \
# ensure that /var/run/mysqld (used for socket and lock files) is writable regardless of the UID our mysqld instance ends up having at runtime
    && chmod 777 /var/run/mysqld

# 设置挂载点
VOLUME /var/lib/mysql

# 拷贝配置文件到容器中
COPY config/ /etc/mysql/
COPY docker-entrypoint.sh /usr/local/bin/

# 产生新的Layer
RUN ln -s usr/local/bin/docker-entrypoint.sh /entrypoint.sh

# 容器启动时指定脚本docker-entrypoint.sh
ENTRYPOINT ["docker-entrypoint.sh"]

# 暴露端口号
EXPOSE 3306 33060

# 容器启动时启动mysql服务
CMD ["mysqld"]

单机容器编排

Docker Compose和容器编排

  • Docker的最佳实践是一个容器只运行一个进程
  • 通常应用服务都不会单个存在，很多时候服务之间是存在依赖性的。例如应用依赖数据库服务和缓存服务，这时就需要对容器进行编排。例如在服务A启动之前需要保证服务B和服务C启动，否者服务A不能正常提供服务，这时我们就需要手动启动多个服务
  • 针对以上情况，如果服务太多的话，启动服务的工作就会变得额外沉重。为了解决这个问题 Docker 提供了 Docker Compose 作为解决方案
  • Docker Compose 允许对容器进行编排并关联服务之间的关系
  • Docker Compose 使用 yaml 进行配置，通过解析 yaml 配置转为 docker 参数来运行组建容器
  • Docker Compose 使用步骤：定义Dockerfile生成镜像、定义yaml、运行docker-compose up命令

安装Docker Compose

  • 官方文档：https://docs.docker.com/compose/install/
  • Docker Compose的安装方式有两种：pip（因为Docker Compose是用Python开发的）、curl

步骤1：下载 docker-compose

sudo curl -L "https://github.com/docker/compose/releases/download/1.23.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

步骤2：授权

sudo chmod +x /usr/local/bin/docker-compose

快速入门

步骤1：创建 docker-compose.yml 文件，注意docker-compose.yml是缺省文件名字

# docker compose版本号
version: '3.3'
services: 
    # 服务名称（自定义）
    tomcat: 
        # dockerfile文件所在的位置
        # build: .  
        # 除了使用dockerfile还可以直接指定镜像,默认使用最新版本
        image: tomcat
        # 端口映射，相当于 -p 8080:8080
        ports:
            - "8080:8080"

步骤2：运行docker-compose.yml

# 后台运行（在docker-compose文件目录下执行）
docker-compose up -d
# 指定文件启动
docker-compose -f docker-compose.yml up -d
# 停止容器
docker-compose stop
# 移除容器
docker-compose rm

下面以 elasticsearch + cerebro + kibana 为例子。因为博主机器内存有限，所以只能单节点做例子，而集群配置可以参考官方文档（注意版本）：

  • https://www.elastic.co/guide/en/elasticsearch/reference/7.5/docker.html

步骤1：扩大宿主机限制，否则 Elasticsearch 可能启动不成功

sudo sysctl -w vm.max_map_count=262144

步骤2：编写 docker-compose.yml 文件

version: "3.6"
services: 
    elasticsearch:
        container_name: elasticsearch
        image: elasticsearch:7.5.0 # 注意需要指定版本号，否则拉取不成功
        ports: 
            - 9200:9200
        environment: 
            - node.name=es-node-1 # 指定节点名称
            - bootstrap.memory_lock=true # 锁定内存大小
            - discovery.type=single-node # 单节点模式
            - "ES_JAVA_OPTS=-Xms512m -Xmx512m" # 配置堆大小
        ulimits: # 覆盖容器原来的限制
            memlock:
                soft: -1
                hard: -1
        restart: always # 重启策略
        networks: # 指定网络
            - elastic     

    kibana:
        container_name: kibana
        image: kibana:7.5.0
        ports: 
            - 5601:5601
        restart: always
        depends_on: 
            - elasticsearch
        networks: 
            - elastic

    # cerebro可用于监控es节点，如内存、cpu等等指标
    cerebro: 
        container_name: cerebro
        image: lmenezes/cerebro
        ports: 
            - 9000:9000
        restart: always
        depends_on: 
            - elasticsearch
        networks: 
            - elastic

# 定义一个网络，用于服务之间使用
networks:
    elastic:
        driver: bridge

搭建私有注册中心

步骤1：拉取docker hub上的registry

docker pull registry

步骤2：构建一个镜像

Dockerfile

FROM scratch
LABEL desc=测试镜像上传到注册中心
CMD echo "hello..."

构建镜像

格式：docker build -t 注册中心所在主机ip:5000/镜像名称 .
docker build -t 192.168.1.102:5000/hello-test .

配置注册中心信任

vim /etc/docker/daemon.json
添加如下内容：(注册中心配置)
{
  "registry-mirrors": ["http://f1361db2.m.daocloud.io"], # 镜像加速
  "insecure-registries": ["192.168.1.102:5000"] # 非SSL源管理镜像
}
vim /lib/systemd/system/docker.service
添加如下内容：(docker服务配置)
EnvironmentFile=-/etc/docker/daemon.json

重启docker服务

systemctl daemon-reload
systemctl restart docker

步骤3：启动注册中心容器

docker run -d -p 5000:5000 116995fd6624

步骤4：将镜像push到私有注册中心

docker push 192.168.1.102:5000/hello-test
