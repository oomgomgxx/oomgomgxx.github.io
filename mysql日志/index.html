InnoDB的redo、redo

redo log

redo log是InnoDB事务日志的组成部分（事务日志 = redo log + 日志缓冲区）。

用于记录数据页更改的物理情况，主要是保证事务的一致性。

与 bin log 的对比：

  • bin log 是 server 层面的日志，用于记录修改数据的操作，在事务提交之前写入磁盘
  • redo log 是 InnoDB 层面的日志，用于记录数据页的物理情况。内容先写入日志缓冲区，随后默认每隔1s刷新到磁盘日志文件中
  • redo log 属于写前日志，即修改操作之前就会先记录，且在事务操作期间不断积累

undo段

  • undo 段主要用来保证事务的回滚，以及作为隔离级别读取一致性的基础
  • 功能和 redo log 相反，用于记录数据页修改前的状态
  • undo 并不位于日志文件，而是在数据库内部的一个内存段中
