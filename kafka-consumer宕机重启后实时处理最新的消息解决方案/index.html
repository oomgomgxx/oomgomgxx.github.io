kafka-consumer宕机重启后实时处理最新的消息解决方案

 2019-03-21   作者: TanDi

方案说明

通过分配随机consumer groupid来达到每次重启后，抹杀掉之前的offset记录

使用

application.yml

#  自定义kafka配置:
kafka:
  consumer:
    groupid0: ${random.long}-topic
    groupid1: ${random.long}-topic
    groupid2: ${random.long}-topic
...

监听程序

    /**
     * 监听故障topic
     * @param json
     */
@KafkaListener(id = "test", topics = "test", groupId = "${kafka.consumer.groupid0}")
public void test(String json) {
    // ...
}
