如何区分网站是静态网站还是动态网站

  1. 拿爬取图片为例，静态网站会把图片直接写入img标签的src中，这个可以通过浏览器右键查看源码观察到；相对静态网站而言，动态网站上的图片是动态加载的，也就是说，它的图片一开始可能只有一小部分（甚至没有）嵌在页面上，随后的图片根据你的浏览程度来给你动态地加载进页面；例如你进行下拉滚动条、点击查看更多等等这些操作来进行异步的内容加载，这就是动态网站（以图片爬虫的角度）；
  2. 强调的是一下，并不是说将图片直接写入img标签的src就是静态网站，上面的例子只是以图片爬取举的例子，针对的是图片爬虫时如何区分图片是静态加载还是动态加载；实质静态网站指的是那种交互操作（页面操作）无需服务器处理的网站，可以用单机游戏来理解这个概念；而动态网站上的交互，如注册这个操作，是会经过服务器来进行数据处理的，这就是动态网站
  3. 目前来说，现在大多数热门的网站都是动态网站；静态网站如我当前这个博客就是静态网站（忽略评论功能）

分析花瓣网图片加载

对于python爬虫而已，分析需要爬取的数据是一个至关重要的步骤，比如按本例子来说，爬取的是花瓣网的图片，而花瓣网是一个动态展示图片的网站，即图片的连接并不是直接编写在img中的src里的，再简单点说就是你不像静态网页那样爬取到你想要的图片链接，所以这时你就需要知道，花瓣网是怎样动态加载图片来展示到页面上的。

首先花瓣网取用下拉时异步加载图片来展示，即一开始打开网页的时候并不是将图片都展示出来，而是当你下拉浏览器滚动条的时候，一部分一部分地加载剩余的图片

验证操作：

  1. 打开浏览器的开发者模式（f12）
  2. 点击network ,再选中XHR（异步请求）
  3. 下拉浏览器并观察

编写爬取图片的代码

因为本身代码量比较少，所以就不进行封装了，一个文件搞定，看起来更加直观


import os

# 图片保存文件
dirName = 'C:/Users/admin/Desktop/py_save'

# 创建文件夹
def mkdir(path):
    path = path.strip()
    isExists = os.path.exists(path)
    if not isExists:
        os.makedirs(path)
        print('文件夹创建成功')
    else:
        print(path, '文件夹已经存在了，不再创建')

# 创建文件夹
mkdir(dirName)

# 保存图片
def saveImg(requests, url, dirName, fileName):
    img = requests.get(url)
    file_name = dirName + '/' + fileName + '.png'
    f = open(file_name, 'ab')
    f.write(img.content)
    print(file_name, '文件保存成功！')
    f.close()


import requests
import _thread
import time
import re
import json

# 通过正则表达式截取到图片信息部分，即app.page["pins"]的内容
re_compile = re.compile('app\.page\["pins"\].*')

# 图片地址拼接用
# https://hbimg.b0.upaiyun.com/  +  图片json['file'\]['key'\] + _fw658
urlPrefix = 'https://hbimg.b0.upaiyun.com/'
urlSuffix = '_fw658'

# 这里我们只爬取10页蝙蝠侠的图片
for i in range(1, 10):

    # 异步请求url样本：https://huaban.com/search/?q=%E8%9D%99%E8%9D%A0%E4%BE%A0&jts5i7zs&page=1&per_page=20&wfl=1
    # 其中%E8%9D%99%E8%9D%A0%E4%BE%A0 是蝙蝠侠的encode之后的结果，重浏览器复制下来就这样，所以并不需要理会

    # 拼接异步请求url来访问花瓣网来获取图片信息
    page = str(i)
    post_param = {"page": page, "per_page": "20", "wfl":'1'}
    url = 'https://huaban.com/search/?q=%E8%9D%99%E8%9D%A0%E4%BE%A0&jts5i7zs'
    response = requests.get(url, params=post_param)
    text = response.text

    datas = re_compile.findall(text)[0]
    split_ = datas.split('= ')[1][:-1]
    loads = json.loads(split_)

    for imgMsg in loads:
        imgName = str(imgMsg['pin_id'])
        imgkey = imgMsg['file']['key']
        imgUrl = urlPrefix + imgkey + urlSuffix
        # print(urlPrefix + imgkey + urlSuffix)
        # 单线程，阻塞下载速度慢
        # saveImg(requests, imgUrl, dirName, imgName)
        # 使用多线程改善效率
        _thread.start_new_thread(saveImg, (requests, imgUrl, dirName, imgName,))
        # 阻塞一下主线程不让它死
        time.sleep(3)
