关于@Conditional注解

@Conditional注解是spring提供的条件注解，用于满足特定条件下才初始化bean的需求时使用；

常用的条件派生注解如下：

条件注解 说明
@ConditionalOnBean 要求上下文中存在特定的bean对象
@ConditionalOnClass 要求类路径中存在指定的class文件
@ConditionalOnExpression 要求满足指定表达式要求
@ConditionalOnMissingBean 要求上下文中不存在特定的bean对象
@ConditionalOnMissingClass 要求类路径中不存在指定的class文件
@ConditionalOnNotWebApplication 要求当前不是一个web应用
@ConditionalOnProperty 要求配置环境中有指定的配置

SpringBoot配置的加载顺序

首先，SpringBoot以main方法启动，所以启动时传入参数的参数优先级别最高；

其次，除了命令行形式传入的参数之外，其实对于很多java应用而言还会到环境变量中获取配置参数，当然SpringBoot也不例外，所以可以将参数配置到环境变量（包含：ide配置文件 > 系统环境变量）；

最后，最低优先级别就是配置文件中的配置了；加载配置文件中的配置这个工作，位于org.springframework.boot.context.config.ConfigFileApplicationListener中完成，加载顺序如下：

  1. 应用同级目录 config 下的配置文件
  2. 应用同级目录下的配置文件
  3. 类路径中 config 下的配置文件
  4. 类路径下的配置文件(既reources下的配置文件)

关于SpringBoot如何弃用默认技术和自动配置

有些时候我们希望使用自己熟悉的技术，如SpringBoot中默认数据库连接池为HikariCP，现在打算使用Druid，那么只需要将HikariCP在pom中将依赖排除（将依赖排除出类路基）再将Druid的starter即可

druid github：https://github.com/alibaba/druid

详细配置：https://github.com/alibaba/druid/tree/master/druid-spring-boot-starter

还有就是有时候我们不希望SpringBoot进行自动配置，如使用多数据源的时候，除了使用@Primary配置方式之外还可以使用排除自动配置改为手动配置来配置多数据源，这时可以在@SpringBootApplication中进行排除自动配置类操作

  • 首先将配置排除
  • 配置文件中添加数据配置
  • 读取配置文件并生产数据源和事务管理器
