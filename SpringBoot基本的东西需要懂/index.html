SpringBoot自动配置的基本原理

初始化自动配置信息

以下为一个简单的springboot启动类

@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

跟中代码到SpringApplication构造函数

public SpringApplication(ResourceLoader resourceLoader, Class<?>... primarySources) {
  this.resourceLoader = resourceLoader;
  this.primarySources = new LinkedHashSet<>(Arrays.asList(primarySources));

  // 当前应用的类型：none、servlet、reactive
  // 默认为none、添加web模块后为servlet、添加webflux模块后为reactive
  this.webApplicationType = WebApplicationType.deduceFromClasspath();

  // 设置初始化器
 setInitializers((Collection)getSpringFactoriesInstances(ApplicationContextInitializer.class));

  // 设置监听器
 setListeners((Collection) getSpringFactoriesInstances(ApplicationListener.class));

  // 设置当前springboot应用的主类，即当前的 DemoApplication.class
 this.mainApplicationClass = deduceMainApplicationClass();
}

可以注意到，其中 设置初始化器 和 设置监听器 调用的是同一个方法 getSpringFactoriesInstances，从方法的名称大概可以得知该方法和获取spring工厂实例有关，方法如下

private <T> Collection<T> getSpringFactoriesInstances(Class<T> type, Class<?>[] parameterTypes, Object... args) {
  // 获取类加载器
  ClassLoader classLoader = getClassLoader();

  // 获取命名（名称）配置
  // Use names and ensure unique to protect against duplicates
  Set<String> names = new LinkedHashSet<>(SpringFactoriesLoader.loadFactoryNames(type, classLoader));

  // 根据名称创建实例
  List<T> instances = createSpringFactoriesInstances(type, parameterTypes, classLoader, args, names);
  AnnotationAwareOrderComparator.sort(instances);
  return instances;
}

根据上面的方法，进一步追中方法 SpringFactoriesLoader.loadFactoryNames(type, classLoader),以下为主要代码

public static List<String> loadFactoryNames(Class<?> factoryClass, @Nullable ClassLoader classLoader) {
  String factoryClassName = factoryClass.getName();
  return loadSpringFactories(classLoader).getOrDefault(factoryClassName,      Collections.emptyList());
}

关于@Conditional注解

@Conditional注解是spring提供的条件注解，用于满足特定条件下才初始化bean的需求时使用；

常用的条件派生注解如下：

条件注解 说明
@ConditionalOnBean 要求上下文中存在特定的bean对象
@ConditionalOnClass 要求类路径中存在指定的class文件
@ConditionalOnExpression 要求满足指定表达式要求
@ConditionalOnMissingBean 要求上下文中不存在特定的bean对象
@ConditionalOnMissingClass 要求类路径中不存在指定的class文件
@ConditionalOnNotWebApplication 要求当前不是一个web应用
@ConditionalOnProperty 要求配置环境中有指定的配置

自定义条件注解

实现Condition接口

public class ConditionalOnHelloWorld implements Condition {
    @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
          // 我这里要求配置中 my.flag 的值为 true 才满足条件
        return Boolean.valueOf(context.getEnvironment().getProperty("my.flag").toLowerCase());
    }
}

使用@Conditional

@Configuration
public class TestConfig {
    @Conditional(ConditionalOnHelloWorld.class)
    @Bean
    public Object test() {
        System.out.println("into TestConfig.test()");
        return new Object();
    }
}

SpringBoot配置的加载顺序

首先，SpringBoot以main方法启动，所以启动时传入参数的参数优先级别最高；

其次，除了命令行形式传入的参数之外，其实对于很多java应用而言还会到环境变量中获取配置参数，当然SpringBoot也不例外，所以可以将参数配置到环境变量（包含：ide配置文件 > 系统环境变量）；

最后，最低优先级别就是配置文件中的配置了；加载配置文件中的配置这个工作，位于org.springframework.boot.context.config.ConfigFileApplicationListener中完成，加载顺序如下：

  1. 应用同级目录 config 下的配置文件
  2. 应用同级目录下的配置文件
  3. 类路径中 config 下的配置文件
  4. 类路径下的配置文件(既reources下的配置文件)

会话级别的H2数据库和ActiveMQ

springboot提供了方便开发者在调研阶段使用的数据库h2和activemq模块，非常方便的是两个模块都并不需要做任何配置就可以马上使用，既无需再本地装数据库和mq，实属方便；

但需要注意的是，h2和activemq都是会话级别的，即重启之后就会抹杀之前的数据；

使用Actuator来观察应用端点

actuator是springboot提供的一个用于监控和管理应用的模块，并提供了：审计、健康信息、数据采集统计等等功能；

官方文档：https://docs.spring.io/spring-boot/docs/2.2.0.M4/reference/html/#production-ready

基本使用

添加依赖

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>

访问/actuator查看有哪些端点

默认情况下只开放以上端点，如果想要开启所有内置端点则需要添加一下配置

# 暴露所有端点
management.endpoints.enabled-by-default=true
management.endpoints.web.exposure.include=*
# 允许端点关闭应用
management.endpoint.shutdown.enabled=true

常用的端点

请求方式 请求路径 描述
GET /beans 返回应用中的所有bean信息
GET /health 监控检测
GET /env 返回系统所有的环境变量
GET /env/{name} 返回具体的环境变量
GET /info 返回系统信息
GET /conditions 返回自动配置的条件信息
POST /shutdown 关闭应用
GET /mappings 返回应用所有的URL映射
GET /metrics 返回当前应用的系统指标（度量）信息，（提供给如promethues）
GET /scheduledtasks 返回应用的定时任务信息
GET /threaddump 返回应用线程转存信息
请求方式 请求路径 描述
GET /heapdump 返回应用当前堆内存的信息
GET /httptrace 返回应用的http请求的追中信息
GET /caches 返回应用的缓存管理对象信息
POST /refresh 重新加载一次配置参数（如配置文件），用于刷新配置

关于SpringBoot如何弃用默认技术和自动配置

有些时候我们希望使用自己熟悉的技术，如SpringBoot中默认数据库连接池为HikariCP，现在打算使用Druid，那么只需要将HikariCP在pom中将依赖排除（将依赖排除出类路基）再将Druid的starter即可

druid github：https://github.com/alibaba/druid

详细配置：https://github.com/alibaba/druid/tree/master/druid-spring-boot-starter

<dependency>
  <groupId>com.alibaba</groupId>
  <artifactId>druid-spring-boot-starter</artifactId>
  <version>1.1.17</version>
</dependency>

还有就是有时候我们不希望SpringBoot进行自动配置，如使用多数据源的时候，除了使用@Primary配置方式之外还可以使用排除自动配置改为手动配置来配置多数据源，这时可以在@SpringBootApplication中进行排除自动配置类操作

  • 首先将配置排除
@SpringBootApplication(exclude = {DataSourceAutoConfiguration.class, 
        DataSourceTransactionManagerAutoConfiguration.class,
        JdbcTemplateAutoConfiguration.class
})
public class Demo2Application {
    public static void main(String[] args) {
        SpringApplication.run(Demo2Application.class, args);
    }
}
  • 配置文件中添加数据配置
  • 读取配置文件并生产数据源和事务管理器
@Bean
@ConfigurationProperties("one.datasource")
public DataSourceProperties oneDataSourceProperties() {
  return new DataSourceProperties();
}

@Bean
public DataSource oneDataSource() {
  DataSourceProperties dataSourceProperties = oneDataSourceProperties();
  return dataSourceProperties.initializeDataSourceBuilder().build();
}

@Bean
@Resource // 根据bean id注入
public PlatformTransactionManager twoTransactionManager(DataSource oneDataSource) {
  return new DataSourceTransactionManager(oneDataSource);
}

// ------------------------------------------------------

@Bean
@ConfigurationProperties("two.datasource")
public DataSourceProperties twoDataSourceProperties() {
  return new DataSourceProperties();
}

@Bean
public DataSource twoDataSource() {
  DataSourceProperties dataSourceProperties = twoDataSourceProperties();
  return dataSourceProperties.initializeDataSourceBuilder().build();
}

@Bean
@Resource // 根据bean id注入
public PlatformTransactionManager oneTransactionManager(DataSource twoDataSource) {
  return new DataSourceTransactionManager(twoDataSource);
}
