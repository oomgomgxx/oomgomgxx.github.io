接口幂等性解决方案笔记
发表于 2018-08-29| 更新于 2020-04-17|编程分布式
评论数:

何为幂等

幂等：不管执行多少次结果都一样。例如 a = 1
非幂等：每次执行结果都不一样。例如 a = a+1

需要搞清楚的事情

  • 幂等和并发是两件事不要搞混
  • 并发引起的是临界同步问题，而幂等是重复操作问题

常见解决方案

  • 乐观锁状态机
    • 加入version字段
    • 插入数据时进行判断，例如 obj.setVersion(1) ，1是固定值，那么只有第一个操作会成功（相当于 update table set age = xxx and version = 1 where version = 0，只有第一次能成功）
  • 防重表
    • 依赖唯一约束，该表不一定是数据库表，只要提供唯一约束的存储设系统即可 ，例如Redis的setnx
    • 操作前先往防重表插入一条数据，成功插入后才操作。那么如果发生非幂等操作，多余的插入是不会成功
  • 防重Token（推荐）
    • 例如客户端在请求处理操作前先向服务方索取一个token（服务端返回token时应将其持久化），然后客户端需要携带该token进行请求操作。服务端通过判断token是否存在，如果存在则先删除token后并处理请求。如果不存在则返回响应提示。因此以上操作只有第一次请求会被处理，因为服务方会在判断token之后删除token以防客户端携带同一个token进行多次操作。
    • 对于持久化token操作而言，可以存储在HttpSession中，如果是分布式系统也可以存储在Redis中。
    • 可以使用全局唯一ID作为token。例如：ZooKeeper事务唯一ID、MongoDB的ObjectID、Twitter的SnowFlake算法、UUID等等。
    • 案例：例如Struts的token标签就是该原理，在加载页面时会为表单生成一个token，在提交表单时一并提交到服务端，然后服务端会将token保存到session中，再利用Filter来过滤多余的提交操作以达到防止表单重复提交的效果
