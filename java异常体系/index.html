PermGen溢出（jdk1.7及以下）

同样一段代码在jdk1.8下运行

  1. 原因是在jdk1.8中，PermGen（永久代）已经被移除，被metaspace（元空间）替代；原来的PermGen实质是堆内存中的一部分，也叫no heap，用于存储：class元信息、方法信息、静态变量、常量池等等
  2. metaspace用于存储class元信息，而相对于PermGen的其他内容将存储到堆中
  3. metaspace位于物理内存，所以受限制于物理内存

Metaspace溢出（jdk1.8）

使用 -XX:MaxMetaspaceSize=64m 限制元空间大小，否者会一直消耗物理内存

ClassNotFoundException

程序通过以下方法加载类，但是没有找到具有指定名称的类的定义：

  • Class 类中的 forName 方法。
  • ClassLoader 类中的 findSystemClass 方法。
  • ClassLoader 类中的 loadClass 方法。

NoClassDefFoundError

当 Java 虚拟机或 ClassLoader 实例试图在类的定义中加载，但无法找到该类的定义时，抛出此异常。

（Java虚拟机在编译时能找到合适的类，而在运行时不能找到合适的类导致的错误）
