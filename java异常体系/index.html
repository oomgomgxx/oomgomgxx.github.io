Java Heap溢出
PermGen溢出（jdk1.7）

同样一段代码在jdk1.8下运行

  1. 原因是在jdk1.8 中 PermGen（永久代）被移除，取而代之的是 Metaspace（元空间）
  2. 原 PermGen 实质是堆内存中的一部分，所以叫 No Heap，用于存储：Class元信息、方法信息、静态变量、常量池等
  3. Metaspace主要用于存储Class元信息和方法信息，而相对于 PermGen 而言其他内容则转移到了堆中
  4. Metaspace默认没有设置内存上限，但受限制于物理（主）内存
Metaspace溢出（jdk1.8）

使用 -XX:MaxMetaspaceSize=64m 限制元空间大小，否者会一直消耗物理内存

  • Class 类中的 forName 方法。

NoClassDefFoundError

当 Java 虚拟机或 ClassLoader 实例试图在类的定义中加载，但无法找到该类的定义时，抛出此异常。

（Java虚拟机在编译时能找到合适的类，而在运行时不能找到合适的类导致的错误）
