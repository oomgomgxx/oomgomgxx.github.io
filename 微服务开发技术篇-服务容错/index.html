分布式分区容错性应对方案

降级

  降级是一种系统出现过载或服务不可用时的应对方案。例如我们可以先对服务进行等级划分，当系统出现过载时就可以根据服务的优先级别来进行丢车保帅操作，既通过关闭非核心服务来保持核心服务的可用性。又或者当A服务不可用时就需要B服务来代替。例如电商系统在下单成功后需要发送手机短信来通知买家，但由于手机短信服务不可用，那么这时可以由原来的发送手机短信改为发送邮件（如果发送邮件都不可行则直接放弃通知操作）

常见的降级手段

容错

  指的是系统在运行过程中容许发生错误。容错其实是一种能力，即能够化解多种错误就可以说容错能力较强，反之则较弱。例如接下来要说的 Hystrix 就供熔断机制来解决服务间调用时发生的异常问题。

   Hystrix 的基本思想是对被调用服务进行可用性评估来判断是否需要开启断路器。断路器的作用是让服务调用能够做到快速失败。例如 Hystrix 认为服务A不可用，那么开启断路器后接下来调用服务A的操作就会快速失败，这样做可以减少无畏的等待以及可以避免服务雪崩现象。

限流

  如果说降级是从服务优先级的角度来考虑应对故障和过载，那么限流则是从用户访问压力的角度来考虑应对故障和过载。

  限流指的是只允许系统能够承受的访问量限度内进来请求处理，超出系统访问能力范围的请求将
被直接丢弃。限流的手动可以划分为两种

  • 基于请求限流：请求阈值、请求频率、接口调用频率

  • 基于资源限流：线程数 、数据库连接数、请求队列长度

请求排队

  限流操作会将系统承受范围之外的请求直接拒绝，但是在某些场景请求是不允许丢失的。例如经常被人谈论的 秒杀系统就可以使用请求排队来缓解请求处理压力。请求排队可以将请求暂时缓存起来，做到 流量削峰 和增大请求处理吞吐量 的效果。例如消息系统 Kafka 就有消息持久化能力。但要注意，请求排队其实是一种降低系统可用性的操作，因为请求并不会马上处理，所以这时用户并不能马上得知自己的请求结果。

关于服务雪崩和断路器

  如图所示，服务雪崩其实是一种正反馈。指的是在调用链中因为某个服务不可用而导致级联不可用，最后甚至导致整个系统不可用的现象。

  那么如何避险服务雪崩呢？最简单的方法就是引入调用超时机制，即尽早地放弃调用从而释放连接资源，以免资源因为长时间被占用而耗尽。除了超时机制之外就是断路器了，所谓的断路器其实就是熔断的实现，它的原理和家里的电闸保险丝一样，即一旦发现服务不可用，断路器就会根据状态评估来阻断依赖该服务的调用者，主要是让调用快速失败来避免不必要的资源消耗。

图解断路器

Netflix Hystrix

基本介绍

Hystrix是Netflix开源的分布式服务容错框架。Hystrix主要为分布式服务提供以下功能：

  • 超时控制
  • 断路器
  • 资源隔离
  • 降级操作
  • 请求缓存 和 请求合并

命令模式实践

Hystrix的功能实现采用命令模式来实现，命令模式可以将服务调用者和任务执行者解耦。Hystrix采用这用方式来解决服务之间的级联故障问题。

命令模式优缺点

  • 可做到调用者和执行者解耦。（CQRC架构就基于命令模式，而这里的Student就是聚合对象）
  • 扩展新命令容易
  • 随着新命令的增多会是系统变得复杂

Hystrix工作流程图

  1. 接收到请求后构建command。command有以下两种：
    • HystrixCommand：依赖的服务返回单个操作结果
      • execute()：同步执行
      • queue()：异步执行
    • HystrixObservableCommand：依赖的服务返回多个操作结果
      • observe()：返回一个可监听对象Observable。可以接收多个返回结果，属于Hot Observable，即只能监听部分行为
      • toObservable() ：功能和 observe() 类似，但属于Cold Observable，即可以监听全部行为
      • 需要说的是 Observable 并不是JDK中的那个，而是 RxJava 中的，RxJava 是一个响应式框架
  2. 判断是否有开启 command 的缓存功能，如果有则直接返回上一次 command 的响应结果。否则进入下一阶段
  3. 判断断路器是否有开启。如果有则直接触发 fallback 操作，否则接着判断线程池或信号量是否已经到达预先设置的阈值，如果已达阈值就使 command 直接失败并进入 fallback 操作，反之继续执行命令
  4. 执行命令过程中需要监控调用依赖服务时是否有发生调用异常或调用超时。如果有就通知断路器统计信息判断是否有必要开启断路器
  5. 断路器开启时执行 fallback 操作也是可能会失败的，当失 fallback 操作失败将会直接会抛出异常。

断路器工作原理

断路器的三种状态

  • CLOSED ：关闭，可以正常调用依赖服务
  • OPEN ：开启，请求调用依赖服务会被快速失败
  • HALF-OPEN ：半开启，允许尝试性地调用已经熔断了的依赖服务

调用隔离

船舱隔离

如图所示，Hystrix 可以将用户请求分配到特定的线程池中进行隔离处理，这种操作也被称为船舱隔离。

船舱隔离在 Hystrix 中的应用

Hystrix中的2种请求处理隔离策略

  • 线程池隔离：
    1. 数量可控且受保护，不受客户端恶意调用影响
    2. 待命线程能够提高请求处理效率
    3. 提供内部并发性
    4. 多线程存在上下文切换问题，会增加系统资源开销
  • 信号量隔离：
    1. 信号量实质就是个计数器，比线程池开销低
    2. 不支持超时和退出

在 Hystrix 中提供了 execution.isolation.strategy 属性来设置隔离策略。

请求合并

Hystrix提供了这对相同 command 的请求合并功能。请求合并可以有效地减少网络连接开销

请求缓存

  • 请求缓存可以在 command 相同时，返回上一次的依赖服务的响应成功的结果，这样可以避免重复操作

  • 从图中可以看到，缓存采用 ThreadLoacl 实现，而ThreadLoacl 是一个线程变量，所以Hystrix的请求缓存范围是在单个请求内才起作用的

  • 一旦引入缓存就可能出现缓存一致性问题，所以需要注意。在这方面 Hystrix 提供了清除缓存的方法，在有必要时可以用来清除缓存

    HystrixRequestCache.getInstance(GETTER_KEY, HystrixConcurrencyStrategyDefault.getInstance()).clear("commandKey");

// HystrixCommand 环绕织入
@Around("hystrixCommandAnnotationPointcut() || hystrixCollapserAnnotationPointcut()")
public Object methodsAnnotatedWithHystrixCommand(final ProceedingJoinPoint joinPoint) throws Throwable {
    // 反射获取到目标方法
    Method method = getMethodFromTarget(joinPoint);

    // 方法不能同时被 HystrixCommand、HystrixCollapser 两个注解标注
    if (method.isAnnotationPresent(HystrixCommand.class) {}

    MetaHolderFactory metaHolderFactory = META_HOLDER_FACTORY_MAP.get(HystrixPointcutType.of(method));

    // 从工厂中获取元数据持有者
    MetaHolder metaHolder = metaHolderFactory.create(joinPoint);

    // 【重】可调用对象，包含fallback和普通command的execute方法
    HystrixInvokable invokable = HystrixCommandFactory.getInstance().create(metaHolder);
    ExecutionType executionType = metaHolder.isCollapserAnnotationPresent() ?
            metaHolder.getCollapserExecutionType() : metaHolder.getExecutionType();

    Object result;
    if (!metaHolder.isObservable()) { // 不是可监听
        result = CommandExecutor.execute(invokable, executionType, metaHolder);
    } else { // 可监听
        result = executeObservable(invokable, executionType, metaHolder);
    }
    return result;
}

HystrixCommandFactory.getInstance().create(metaHolder)

// 这里的 HystrixInvokable 实质是一个 GenericCommand
public HystrixInvokable create(MetaHolder metaHolder) {
    HystrixInvokable executable;
    if (metaHolder.isCollapserAnnotationPresent()) { // 是否合并
        executable = new CommandCollapser(metaHolder);
    } else if (metaHolder.isObservable()) { // 是否可观察
        executable = new GenericObservableCommand(HystrixCommandBuilderFactory.getInstance().create(metaHolder));
    } else { // 普通command
        //【重】这里传入了 MetaHolder 用来创建了一个 GenericCommand
        executable = new GenericCommand(HystrixCommandBuilderFactory.getInstance().create(metaHolder));
    }
    return executable;
}

然后看 GenericCommand 类

public class GenericCommand extends AbstractHystrixCommand<Object> {
    public GenericCommand(HystrixCommandBuilder builder) {
        super(builder);
    }
    // 执行正常逻辑
    @Override
    protected Object run() throws Exception {
        return process(new Action() {
            @Override
            Object execute() {
                return getCommandAction().execute(getExecutionType());
            }
        });
    }
    // 执行fallback
    @Override
    protected Object getFallback() {
        final CommandAction commandAction = getFallbackAction();
        if (commandAction != null) {
            return process(new Action() {
                @Override
                Object execute() {
                    MetaHolder metaHolder = commandAction.getMetaHolder();
                    Object[] args = createArgsForFallback(metaHolder, getExecutionException());
                    return commandAction.executeWithArgs(metaHolder.getFallbackExecutionType(), args);
                }
            });
        } else {
            // Fallback失败会进入父类Fallback
            return super.getFallback();
        }
    }
}

命令执行器 - CommandExecutor

    public static Object execute(HystrixInvokable invokable, ExecutionType executionType, MetaHolder metaHolder) throws RuntimeException {
        switch (executionType) {
            case SYNCHRONOUS: { // 同步执行
                // 转换成可执行状态
                // 执行 HystrixCommand 的execute()方法
                return castToExecutable(invokable, executionType).execute();
            }
            case ASYNCHRONOUS: { // 异步执行
                HystrixExecutable executable = castToExecutable(invokable, executionType);
                if (metaHolder.hasFallbackMethodCommand()
                        && ExecutionType.ASYNCHRONOUS == metaHolder.getFallbackExecutionType()) {
                    return new FutureDecorator(executable.queue());
                }
                return executable.queue();
            }
            case OBSERVABLE: { // 可监听执行
                HystrixObservable observable = castToObservable(invokable);
                return ObservableExecutionMode.EAGER == metaHolder.getObservableExecutionMode() ? observable.observe() : observable.toObservable();
            }
            default:
                throw new RuntimeException("unsupported execution type: " + executionType);
        }
    }

    public static class Factory {
        private static ConcurrentHashMap<String, HystrixCircuitBreaker> circuitBreakersByCommand = new ConcurrentHashMap<String, HystrixCircuitBreaker>();
        public static HystrixCircuitBreaker getInstance(HystrixCommandKey key, HystrixCommandGroupKey group, HystrixCommandProperties properties, HystrixCommandMetrics metrics) {
            HystrixCircuitBreaker previouslyCached = circuitBreakersByCommand.get(key.name());
            if (previouslyCached != null) {
                return previouslyCached;
            }
            // putIfAbsent有则返回value，无则添加且返回null
            HystrixCircuitBreaker cbForCommand = circuitBreakersByCommand.putIfAbsent(key.name(), new HystrixCircuitBreakerImpl(key, group, properties, metrics));
            if (cbForCommand == null) {
                // 表示put成功
                return circuitBreakersByCommand.get(key.name());
            } else {
                // 表示先前已经有了
                return cbForCommand;
            }
        }
        // 根据HystrixCommandKey获取断路器
        public static HystrixCircuitBreaker getInstance( key) {
            return circuitBreakersByCommand.get(key.name());
        }
        static void reset() {
            circuitBreakersByCommand.clear();
        }
    }

    static class HystrixCircuitBreakerImpl implements HystrixCircuitBreaker {
        private final HystrixCommandProperties properties;
        private final HystrixCommandMetrics metrics;
        // 断路器开关
        private AtomicBoolean circuitOpen = new AtomicBoolean(false);
        private AtomicLong circuitOpenedOrLastTestedTime = new AtomicLong();
        protected HystrixCircuitBreakerImpl(HystrixCommandKey key, HystrixCommandGroupKey commandGroup, HystrixCommandProperties properties, HystrixCommandMetrics metrics) {
            this.properties = properties;
            this.metrics = metrics;
        }
        // 尝试关闭断路器
        public void markSuccess() {
            if (circuitOpen.get()) {
                if (circuitOpen.compareAndSet(true, false)) {
                    metrics.resetStream();
                }
            }
        }
        @Override
        public boolean allowRequest() {
            if (properties.circuitBreakerForceOpen().get()) {
                return false; // 如果断路器已经打开了则不执行command
            }
            // 断路器没有打开，执行command
            if (properties.circuitBreakerForceClosed().get()) {
                isOpen();
                return true;
            }
            // 半打开状态
            return !isOpen() || allowSingleTest();
        }
        // 半打开状态，放行下一个commmand尝试调用依赖服务
        public boolean allowSingleTest() {
            // 获取断路器打开时的时间戳
            long timeCircuitOpenedOrWasLastTested = circuitOpenedOrLastTestedTime.get();
            // 必须休眠时间circuitBreakerSleepWindowInMilliseconds，默认5秒
            if (circuitOpen.get() && System.currentTimeMillis() > timeCircuitOpenedOrWasLastTested + properties.circuitBreakerSleepWindowInMilliseconds().get()) {
                if (circuitOpenedOrLastTestedTime.compareAndSet(timeCircuitOpenedOrWasLastTested, System.currentTimeMillis())) {
                    return true; // 尝试成功
                }
            }
            return false; // 尝试失败
        }

        @Override
        public boolean isOpen() {
            if (circuitOpen.get()) {
                return true;// 处于打开状态
            }
            HealthCounts health = metrics.getHealthCounts();
            if (health.getTotalRequests() < properties.circuitBreakerRequestVolumeThreshold().get()) {
                // 请求总数低于配置circuitBreakerRequestVolumeThreshold（默认20），表示未打开
                return false;
            }
            if (health.getErrorPercentage() < properties.circuitBreakerErrorThresholdPercentage().get()) {
                // 请求错误百分比低于配置circuitBreakerErrorThresholdPercentage（默认50），表示未打开
                return false; 
            } else {
                // 打开断路器
                if (circuitOpen.compareAndSet(false, true)) {
                    // 记录断路器打开时的时间戳
                    circuitOpenedOrLastTestedTime.set(System.currentTimeMillis());
                    return true;
                } else {
                    return true;
                }
            }
        }

    }
}

abstract class AbstractCommand<R> implements HystrixInvokableInfo<R>, HystrixObservable<R> {

    protected final HystrixCircuitBreaker circuitBreaker;

详细使用

相关注解

@HystrixCommand
  • commandKey ：命令标识
  • groupKey ：命令组标识
  • commandProperties ：命令属性
  • threadPoolKey ：线程池标识
  • threadPoolProperties ：线程池属性
  • fallbackMethod ：回退方法
  • defaultFallback ：默认回退方法，和fallbackMethod一样但方法不能有参数
  • ignoreExceptions ：忽略异常（不触发fallback）
  • observableExecutionMode ：执行RxJava的Observabl执行模式，默认 ObservableExecutionMode.EAGER标识马上执行，而LAZY则是有订阅者时才执行
@HystrixCollapser
  • collapserKey：合并标识
  • scope：操作域。默认为Scope.REQUEST，即请求级别
  • collapserProperties：合并属性
  • batchMethod：批处理命令的方法名称
@HystrixProperty
  • name：属性名称
  • value：属性值

Feign中使用Hystrix

@FeignClient

  • value：服务名称
  • name：与value功能一致
  • qualifier：FeignClient限定名称。在多个同类型FeignClient时以作区别
  • decode404：404解码，例如信息解码成json
  • configuration：FeignClient独立配置类，会覆盖全局配置。可以自定义Encoder、Decoder、LogLevel、Contract
  • fallback ：指定回退类，需要实现FeignClient
  • fallbackFactory：指定回退工厂类，功能与fallback一样，但可以获取回退原因，需要实现FallbackFactory接口

在Feign中开启Hystrix功能

feign:
  hystrix:
    enabled: true

实现 fallbackFactory 来支持降级

@Slf4j
@Component
public class UserFeignClientFallbackFactory implements FallbackFactory<UserFeignClient> {

    @Override
    public UserFeignClient create(Throwable cause) {
        return new UserFeignClient() {

            @Override
            public List selectAllUser() {
                log.info("cause -> {}", cause);
                return null;
            }

            @Override
            public User deleteUserById(Long id) {
                log.info("cause -> {}", cause);
                return null;
            }

            @Override
            public User addUser(Long id, String name, int age) {
                log.info("cause -> {}", cause);
                return null;
            }
        };
    }
}

FeignClient 接口

@FeignClient(value = "MSA-EUREKA-PROVIDER-USER", fallbackFactory = UserFeignClientFallbackFactory.class)
public interface UserFeignClient {

    @GetMapping("/user/find/all")
    List selectAllUser();

    /**
     * 需要注意的是，只要使用了Feign并启用Hystrix
     * Hystrix就会为每个FeignClient方法配置一个断路器
     * 即不用直接编写@HystrixCommand，如果添加上@HystrixCommand会抛异常
     * 参考：https://cloud.spring.io/spring-cloud-openfeign/reference/html/#spring-cloud-feign-hystrix
     */
    @DeleteMapping("/user/delete/{id}")
    User deleteUserById(@PathVariable Long id);

    @PostMapping("/user/add")
    User addUser(@RequestParam Long id,
                  @RequestParam String name,
                  @RequestParam int age);
}

图中的信息（右上角是说明）：

  • Success：深绿色，请求成功
  • Short-Circuited：蓝色，请求短路（断路器开启，请求被快速失败）
