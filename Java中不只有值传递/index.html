Java中不只有值传递

 2020-04-03   作者: TanDi   类型：原创

论述

  1. Java语言层面是屏蔽了指针的所以不存在引用传递。

  2. 网上一般的论述是拷贝了对象的地址所以是值传递，但这种说法是站不住脚的，因为这和值传递的概念不符。值传递的特点是通过拷贝值来隔离两个不用的栈帧域，让其在操作期间互不干扰。但在Java中普通的引用类型并不如是，因此实质也不是值传递。

  1. 准确点来说，Java的基本数据类为值传递，而引用类型则为共享对象传递。

  2. String引用类型在Java中有着特殊实现。特点是不可变，且有运行时字符串常量池来提高字符串的利用率。因为字符串在程序中的出现率颇高，为了提高利用率才使用池来维护，其次因为池是线程共享的，所以为了避免出现并发修改带来同步问题的情况发生而设计成不可变。除此之外，字符串字面量本身都有与之对应的String对象实例，而且可以是1到多个，所以传递String本质还是传递共享对象，只不过是字符串字面量的指向没变而已。

图片源自维基百科

总结

个人理解网上之所以总是在值传递和引用传递之间徘徊是因为被概念所限制了思想（毕竟国人搞概念还是世界首屈一指的）。个人认为不论是值传递也好，引用传递也好，还是共享对象传递也好，都不重要，因为本质上它们只是对变量传递过程的一种描述而已，没没必要过度解读。解决问题应该了解其本质，譬如我们只需要知道在Java中普通的引用实例在栈帧中的操作可能会相互影响，这就足够了。
