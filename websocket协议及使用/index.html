了解WebSocket协议

 2018-04-12   作者: TanDi   类型：原创

内容修订

是什么

了解更多：https://tools.ietf.org/html/rfc6455

  • 是一种应用层通信协议。可通过单个TCP连接进行全双工通信
  • 为了兼容HTTP协议，目前版本的 WebSocket 会从 HTTP 之上进行升级转变为 WebSocket（TCP握手成功之后）

知识拓展

  1. 什么是单工通信、半双工通信、全双工通信
    • 单工通信 - 信息只能单向传送
    • 半双工通信 - 信息能双向传送，但不能同时双向传送
    • 全双工通信 - 信息能够同时双向传送
  1. OSI模型

为什么需要

HTTP协议的缺点

HTTP是一种无状态（通信过程不会记录状态记录，是一次性的）、单向（只能通过请求-应答方式，不能主动推送）的应用层通信协议，所以如果想要实现服务器推送功能，就需要客户端使用 XMLHttpRequest 或 <iframe> 等技术主动轮询来代替实现。但这样地频繁请求，不但效率不高而且还消耗无畏的网络资源，但如果使用 WebSocket 的话，就可以很简单地实现这样的功能。因为 WebSocket 只需要通过一条 TCP 连接就可以实现全双工通信，对于一些客户端和服务器需要进行频繁互动的操作显然 WebSocket 的优势更大。

HTTP的短连接和长连接的区别

HTTP协议本身是基于TCP来通信的，在1.1版本前HTTP默认使用短连接的方式来通信，意思是每个请求都需要进行三次TCP握手，而通信结束后就断开，TCP连接不会被复用。这样的好处是能够减少不必要的网络传输消耗，而且在2G通信年代，这样的通信方式已经能够满足使用的需求。但随着3G、4G的到来，Web网站的内容也不再是简单的文字和小图片，而还有视频、音频、大图片等，所以在1.1版本开始，HTTP从默认的短连接该为了长连接（默认请求头connection:keep-alive）,即单个TCP连接可以处理多个HTTP请求，从而很大程度地减少了TCP握手过程带来的消耗，使得请求响应更快。

握手过程

WebSocket 连接会以 HTTP 连接开始自己的生命周期，而这个过程称为 WebSocket 握手。

1）浏览器向服务器发送请求，通过 Upgrade 请求头表明希望升级为 WebSocket 通信方式

GET ws://echo.websocket.org/?encoding=text HTTP/1.1
Origin: http://websocket.org
Cookie: __utma=99as
Connection: Upgrade
Host: echo.websocket.org
Sec-WebSocket-Key: uRovscZjNol/umbTt5uKmw==
Upgrade: websocket
Sec-WebSocket-Version: 13

2）只要服务器支持 WebSocket 协议就会执行升级操作。默认情况下 Websocket 使用与 HTTP（80）和 HTTPS（443） 相同的端口

HTTP/1.1 101 WebSocket Protocol Handshake
Date: Fri, 10 Feb 2012 17:38:18 GMT
Connection: Upgrade
Server: Kaazing Gateway
Upgrade: WebSocket
Access-Control-Allow-Origin: http://websocket.org
Access-Control-Allow-Credentials: true
Sec-WebSocket-Accept: rLHCkw/SKsO9GAH/ZSFhBATDKrU=
Access-Control-Allow-Headers: content-type

WebSocket和Server Sent Event（SSE）的区别

主要区别

  1. 都可以实现服务端推送，并且都是长连接
  2. SSE仍然使用HTTP协议进行通讯，即通信会采用HTTP报文
  3. WebSocket可以进行双向通讯，但SSE只能单向，即服务端推送到客户端

WebSocket快速入门案例

Spring + H5

前端

无需导入插件，因为 websocket 是 HTML5 规范中的内容，目前普遍的浏览器都支持

<!DOCTYPE HTML>
<!-- js脚本 -->
<script>

    //创建websocket对象
    var websocket = new WebSocket("ws://localhost:8888/websocket/ws");
    console.info(websocket);

    //连接成功后触发
    websocket.onopen = function(){
        console.info("连接成功!!!");
        websocket.send("服务端你好，我是客户端呢");
    }

    //接收到服务器响应时触发
    websocket.onmessage = function(res){
        console.info(res);
    }

    //连接关闭时触发
    websocket.onclose = function() { 
      alert("Connection is closed..."); 
   };

</script>

自定义WebSocketHandler

@Component
public class MyWebSocketHandler extends TextWebSocketHandler {

    // 建立连接后执行
    @Override
    public void afterConnectionEstablished(WebSocketSession session) throws Exception {
        System.out.println("连接成功！！");
        // 随便向客户端发送一些内容
        for(int i=0; i<5; i++) {
            TimeUnit.SECONDS.sleep(5);
            session.sendMessage(new TextMessage("客户端你好，我是服务端"));
        }
    }
}

配置WebSocket

@Configuration
@EnableWebSocket
public class WebSocketConfig implements WebSocketConfigurer {

    @Autowired
    private WebSocketHandler webSocketHandler;

    // 注册WebSocket处理器
    @Override
    public void registerWebSocketHandlers(WebSocketHandlerRegistry registry) {
        registry.addHandler(webSocketHandler, "/ws").setAllowedOrigins("*");
    }
}

@ServerEndpoint("/ws")
@Component
public class MyWebSocketServer {

    @OnOpen
    public void onopen(Session session) throws Exception {
        System.out.println("连接成功");
        session.getBasicRemote().sendText("客户端你好啊，我是websocket服务端");

Spring + SockJS + STOMP协议

  • SockJS 可以支持哪些不能使用 WebSocket 的浏览器

  • 采用 Spring + SockJS + STOMP协议 方案，可以提高 WebSocket 通信的可定制程度

前端

Thymeleaf模板

<!DOCTYPE html>
<html lang="zh_CN" xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>index</title>
    <!-- sockjs + stomp -->
    <script src="https://cdn.bootcss.com/sockjs-client/1.1.4/sockjs.min.js"></script>
    <script src="https://cdn.bootcss.com/stomp.js/2.3.3/stomp.min.js"></script>
</head>
<body onload="disconnect()">

    <script type="text/javascript">
        var stompClient = null;
        function setConnected(connected) {
            document.getElementById('connect').disabled = connected;
            document.getElementById('disconnect').disabled = !connected;
            document.getElementById('conversationDiv').style.visibility = connected ? 'visible' : 'hidden';
            document.getElementById('response').innerHTML = '';
        }
        function connect() {
            // 可以看到SockJS可以采用的是HTTP协议进行连接，这是为了兼用不支持WebSocket的浏览器
            var url = 'http://localhost:8080/thymeleaf_demo/chat';
            var socket = new SockJS(url);

            stompClient = Stomp.over(socket);
            stompClient.connect({}, function () {
                // 打开输入框，因为已经连接成功
                setConnected(true);
                // 订阅MQ消息端点
                stompClient.subscribe('/topic/messages', function (response) {
                    showMessageOutput(JSON.parse(response.body));
                });

                // stompClient.subscribe('/app/chat', function (response) {
                //     showMessageOutput(JSON.parse(response.body));
                // });

            });
        }

        function disconnect() {
            if (stompClient != null) {
                stompClient.disconnect();
            }
            setConnected(false);
            console.log("Disconnected");
        }

        function sendMessage() {
            console.info("send message")
            var from = document.getElementById('from').value;
            var text = document.getElementById('text').value;
            var send = JSON.stringify({'from': from, 'text': text});
            // 往服务端发送消息
            stompClient.send("/app/chat", {}, send);
        }

        function showMessageOutput(messageOutput) {
            var response = document.getElementById('response');
            var p = document.createElement('p');
            p.style.wordWrap = 'break-word';
            p.appendChild(document.createTextNode(messageOutput.from + ": "
                + messageOutput.text + " (" + messageOutput.time + ")"));
            response.appendChild(p);
        }
    </script>

    <div>
        <div>
            <input type="text" id="from" placeholder="Choose a nickname" value="张三"/>
        </div>
        <br/>
        <div>
            <button id="connect" onclick="connect();">Connect</button>
            <button id="disconnect" disabled="disabled" onclick="disconnect();">

配置

@Configuration
@EnableWebSocketMessageBroker
public class WebSocketStompConfig implements WebSocketMessageBrokerConfigurer {

    @Override
    public void configureMessageBroker(MessageBrokerRegistry registry) {
        registry.enableSimpleBroker("/topic"); // 消息代理(MQ)前缀
        registry.setApplicationDestinationPrefixes("/app"); // 当前消息代理“目标”前缀，用于客户端发送消息时使用
    }

    @Override
    public void registerStompEndpoints(StompEndpointRegistry registry) {
        registry.addEndpoint("/chat");
        registry.addEndpoint("/chat").withSockJS();
        // 其中一个使用 sockjs 是为了“弹性”，即客户端可用 sockjs 也可以不用
    }
}

使用

@Controller
public class TestController {

    @GetMapping("/")
    public String test(Model model) {
        model.addAttribute("name", "Sam");
        model.addAttribute("age", 18);
        return "index";
    }

    private SimpMessagingTemplate simpMessagingTemplate;
    public TestController(SimpMessagingTemplate simpMessagingTemplate) {
        this.simpMessagingTemplate = simpMessagingTemplate;
    }

    @ResponseBody
    @GetMapping("/send")
    public void sendMessage() {
        String time = new SimpleDateFormat("HH:mm").format(new Date());
        OutputMessage message = new OutputMessage("服务端", "sendMessage处理器", time);
        simpMessagingTemplate.convertAndSend("/topic/messages", message);
    }

    @MessageMapping("/chat") // 表示这是一个端点处理器,用于接收客户端发送过来的stomp消息
    @SendTo("/topic/messages") // 将return发送到MQ的"/topic/messages"
    public OutputMessage send(Message message) throws Exception {
        System.out.println(message);
        String time = new SimpleDateFormat("HH:mm").format(new Date());
        return new OutputMessage("服务端", "send处理器", time);
    }

    /**
     * Stomp客户端订阅时触发
     * 特别注意，以当前案例为例子 @SubscribeMapping 只能处理 /app 前缀的请求
     */
//    @SubscribeMapping("/chat")
