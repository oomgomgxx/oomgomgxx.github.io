公共基础服务之Prometheus监控入门

应用的监控方式有那些？

Logging：系统日志，最常用的监控手段，特点是结构化、离散、文本形式

Tracing：请求链追踪，监控微服务架构下请求在多个节点之间的调用流程

Metrics：度量/指标，即单位时间内应用系统的指标，支持聚合运算、图形展示

Healthchecks：健康检查，专门用于检查应用功能是否可用（健康）的endpoint

简单了解一下时间序列数据库

  • 时间序列数据，通俗点说就是某个时间点所产生的数据
  • 时间序列数据库指的就是用于保存一系列有序的时间点指标数据的数据库
  • 比较流行的时间序列数据库有如InfluxDB、RRDtool、Kdb+、Prometheus

关于Prometheus

  • 一款开源监控(Metrics)工具，本质是 golang 实现的一款时间序列数据库（TSDB）
  • 支持多维度监控（标签，既即使Metrics一样但标签不同的话也是不同的时间序列数据）和Metrics的拉模式
  • 支持告警功能
  • 社区活跃，有各种的exporters（相当与agent，作用是无需修改被监控系统的情况下获取起Metrics信息）
  • 单机性能优异，每秒可消费百万级别的时间序列和上千的targets（监控目标）

Prometheus架构

以下为官网给出的架构图

从架构图中可以简单了解到如下内容：

  • 支持 pull 形式获取 metrics 数据，如拉取 exporters、pushgateway
  • 除了支持将监控目标配置到本地之外还支持服务发现的形式来动态获取监控目标。如（DNS、Consul、K8s）
  • 支持当某个指标到得设定阀值时将告警信息推送给Alertmanager（主要作用是将告警信息分类去重以及转发到特定的目标，如email、钉钉）
  • 支持使用 PromQL 来查询 Prometheus 所监控的系统应用的指标
  • 支持不同的UI界面，如自带的 Prometheus web UI 还有比较流行的 Grafana 并支持 API 形式获取监控指标
  • 通过 TSDB 将时间序列数据持久化到磁盘上

Prometheus支持的Metrics种类

  • Counter(计数器)：
    • 始终增加
    • http请求数、浏览数等等
  • Gauge(测量仪)
    • 当前值的一个快照测量，可增加可减少
    • 磁盘使用率、用户在线人数
  • Histogram(直方图/柱状图)
    • 通过分桶（bucket）方式统计样本
    • 分析数据分布情况
  • Summary(汇总统计)
    • 根据样本统计出百分比
    • 分析数据分布情况

1）metrics说明

以prometheus的 /metrics下的一个metrics为例子

prometheus_http_request_duration_seconds_bucket

  • metrics名称

{handler=”/“,le=”0.1”}：

  • 其中handler和le在prometheus称之为标签（维度）

2）metrics采集方式

prometheus支持两种pull方式：

  • 直接采集：如SpringBoot中的Actuator或prometheus模块可以直接提供metrics的采集endpoint
  • 间接采集：如MySQL一样已经成型的产品，可以通过对用的exporters来进行简介采集
