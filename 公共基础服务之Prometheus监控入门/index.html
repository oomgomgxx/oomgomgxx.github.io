公共基础服务之Prometheus监控入门

 2019-02-21   作者: TanDi

应用的监控方式有那些？

Logging：系统日志，最常用的监控手段，特点是结构化、离散、文本形式

Tracing：请求链追踪，监控微服务架构下请求在多个节点之间的调用流程

Metrics：度量/指标，即单位时间内应用系统的指标，支持聚合运算、图形展示

Healthchecks：健康检查，专门用于检查应用功能是否可用（健康）的endpoint

简单了解一下时间序列数据库

  • 时间序列数据，通俗点说就是某个时间点所产生的数据
  • 时间序列数据库指的就是用于保存一系列有序的时间点指标数据的数据库
  • 比较流行的时间序列数据库有如InfluxDB、RRDtool、Kdb+、Prometheus

关于Prometheus

  • 一款开源监控(Metrics)工具，本质是golang实现的一款时间序列数据库（TSDB）
  • 支持多维度监控（标签，既即使Metrics一样但标签不同的话也是不同的时间序列数据）和Metrics的拉模式
  • 支持告警功能
  • 社区活跃，有各种的exporters（相当与agent，作用是无需修改被监控系统的情况下获取起Metrics信息）
  • 单机性能优异，每秒可消费百万级别的时间序列和上千的targets（监控目标）

Prometheus架构

以下为官网给出的架构图

从架构图中可以简单了解到如下内容：

  • 支持pull形式获取metrics数据，如拉取exporters、pushgateway
  • 除了支持将监控目标配置到本地之外还支持服务发现的形式来动态获取监控目标。如（DNS、Consul、K8s）
  • 支持当某个指标到得设定阀值时将告警信息推送给Alertmanager（主要作用是将告警信息分类去重以及转发到特定的目标，如email、钉钉）
  • 支持使用PromQL来查询Prometheus所监控的系统应用的指标
  • 支持不同的UI界面，如自带的Prometheus web UI还有比较流行的Grafana并支持API形式获取监控指标
  • 通过TSDB将时间序列数据持久化到磁盘上

Prometheus支持的Metrics种类

  • Counter(计数器)：
    • 始终增加
    • http请求数、浏览数等等
  • Gauge(测量仪)
    • 当前值的一个快照测量，可增加可减少
    • 磁盘使用率、用户在线人数
  • Histogram(直方图/柱状图)
    • 通过分桶（bucket）方式统计样本
    • 分析数据分布情况
  • Summary(汇总统计)
    • 根据样本统计出百分比
    • 分析数据分布情况

关于metrics和metrics采集方式

1）metrics说明

以prometheus的 /metrics下的一个metrics为例子

prometheus_http_request_duration_seconds_bucket{handler="/",le="0.1"}

prometheus_http_request_duration_seconds_bucket：metrics名称

{handler=”/“,le=”0.1”}：其中handler和le在prometheus称之为标签（维度）

2）metrics采集方式

prometheus支持两种pull方式：

  1. 直接采集：如SpringBoot中的Actuator或prometheus模块可以直接提供metrics的采集endpoint
  2. 间接采集：如MySQL一样已经成型的产品，可以通过对用的exporters来进行简介采集

prometheus.yml配置文件

# 全局参数配置
global:
  # 需要注意的是，如果刷新和统计频率过于频繁会对当前机器造成性能问题
  scrape_interval: 15s # 刷新频率
  evaluation_interval: 15s # 统计频率

# 告警配置
# Alertmanager configuration
alerting:
  alertmanagers:
  - static_configs:
    - targets:
      # - alertmanager:9093

# 监控规则
# Load rules once and periodically evaluate them according to the global 'evaluation_interval'.
rule_files:
  # - "first_rules.yml"
  # - "second_rules.yml"

# 监控目标配置
scrape_configs:
  # The job name is added as a label `job=<job_name>` to any timeseries scraped from this config.
  - job_name: 'prometheus'
      # 默认访问目标uri'/metrics'
    # metrics_path defaults to '/metrics'
    # 默认使用协议 'http'
    # scheme defaults to 'http'.
    static_configs:
    - targets: ['localhost:9090']

启动prometheus，访问 localhost:9090

./prometheus

监控例子 - MySQL监控

在上面已经提到过，prometheus支持两种metrics的采集方式，一种是直接采集、一种是间接采集;

因为MySQL是一个成型的产品，所以这里采用间接采集metrics，即需要使用exporter;

MySQL的exporter下载地址：https://github.com/prometheus/mysqld_exporter/releases

1）解压如下：

➜  mysqld_exporter-0.12.1.linux-amd64 ll
总用量 15M
-rw-r--r-- 1 tandi tandi 12K  7月 29 20:47 LICENSE
-rwxr-xr-x 1 tandi tandi 15M  7月 29 20:36 mysqld_exporter
-rw-r--r-- 1 tandi tandi  65  7月 29 20:47 NOTICE

2）在mysqld_exporter根目录添加用于访问MySQL的配置文件

[client]
# mysql所在ip
host=0.0.0.0
# mysql用户密码
user=root 
password=123456

4）在prometheus配置文件中将mysqld_exporter配置为监控目标

# 监控目标配置
scrape_configs:
  # The job name is added as a label `job=<job_name>` to any timeseries scraped from this config.
  - job_name: 'prometheus'
      # 默认访问目标uri'/metrics'
    # metrics_path defaults to '/metrics'
    # 默认使用协议 'http'
    # scheme defaults to 'http'.
    static_configs:
    - targets: ['localhost:9090']

  # ########################  MySQL监控 ########################
  - job_name: 'mysql'
    # scrape_interval: 5s
    static_configs:
    # 9104是Exporter端口
    - targets: ['localhost:9104']

5）重启prometheus访问 http://localhost:9090/targets 如果mysql为up表示已经配置成功，可通过http://localhost:9090/graph进行PromQL进行指标查询
