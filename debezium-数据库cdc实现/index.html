使用Debezium实现数据库的CDC功能

 2018-12-17   作者: TanDi

是什么？

  • debezium是一个开源的分布式数据库事件流工具
  • debezium的功能类似于数据库的trigger，且比trigger功能强大
  • 可以用它来实现cdc功能（change data capture,CDC）

关于Debezium Connector for MySQL

  • debezium连接器通过观察mysql二进制日志的变动来生成操作事件，每个表中发生的所有事件数据都会“流动”到kafka对应的topic上

  • 连接器会在第一次连接mysql数据库的时候生成一个快照，完成后会从生成快照的那个点开始读取二进制日志，这样做的目的有两个：

    1）生成快照的原因：mysql配置（my.ini）存在过时清除设置，即可能会在某个时候二进制日志文件内容会被mysql清除

    2）从生成快照的那个点开始观察：从生成点读而不从生成后的点读原因是，预防在生成快照过程中数据库执行了修改操作

  • 连接器若中途故障，那么下次重启，会在故障点开始读取二进制日志；生成快照故障则是重新生成

下载Debezium和Confluent

Debezium：https://repo1.maven.org/maven2/io/debezium/debezium-connector-mysql/

Confluent：https://www.confluent.io/connector/debezium-mysql-cdc-connector/

简单说明：

  • debezium-connector-mysql：提供mysql二进制日志文件变动监听服务
  • debezium-mysql-cdc-connector：Confluent套件，提供CDC相应功能（自带kafka、zookeeper、avro等等功能），特别重要的RESTful server

配置方式1：使用Debezium + Confluent

注意：配置好JDK环境下进行

server.properties（kafka配置文件)：修改或配置

# 192.168.4.161为kafka所在主机ip
listeners=PLAINTEXT://192.168.4.161:9092
advertised.listeners=PLAINTEXT://192.168.4.161:9092

schema-registry.properties（confluent schema-registry组件配置文件)

listeners=http://0.0.0.0:8081
# zookeeper
kafkastore.connection.url=192.168.4.161:2181
# kafka
kafkastore.bootstrap.servers=PLAINTEXT://192.168.4.161:9092

connect-avro-distributed.properties（confluent connect avro格式连接配置文件)**

# kafka
bootstrap.servers=192.168.4.161:9092
# kafka connect 从 source读取数据是的加密格式，使用avro正反序列化，confluent提供
key.converter=io.confluent.connect.avro.AvroConverter
key.converter.schema.registry.url=http://192.168.4.161:8081
value.converter=io.confluent.connect.avro.AvroConverter
value.converter.schema.registry.url=http://192.168.4.161:8081
# 提供REST API服务
rest.host.name=192.168.4.161
rest.port=8083
# CDC需要到的依赖包
# share/java/* ：confluent提供
# /usr/local/debezium-connector-mysql/*：debezium插件提供
plugin.path=share/java/*,/usr/local/debezium-connector-mysql/*

profile（linux环境变量）：注意插件变量名称一定要定义在CLASSPATH中，因为confluent提供的avro相关组件中的shell文件中使用到CLASSPATH

# user settings
JAVA_HOME=/usr/local/jdk1.8.0_201
NODE_HOME=/usr/local/node-v10.15.0-linux-x64
PATH=$JAVA_HOME/bin:$JAVA_HOME/jre/bin:$NODE_HOME/bin:$PATH
CLASSPATH=/usr/local/debezium-connector-mysql/*
export CLASSPATH
export JAVA_HOME PATH NODE_HOME

启动测试

步骤1：启动zookeeper（这里用的是独立的zookeeper，没有confluent自带的）

bin/zkServer.sh start ../confluent-5.1.0/etc/kafka/zookeeper.properties

步骤2：启动kafka-server

bin/kafka-server-start etc/kafka/server.properties

步骤3：启动schema-registry server

bin/schema-registry-start etc/schema-registry/schema-registry.properties

步骤4：启动kafka-connect

bin/connect-distributed etc/schema-registry/connect-avro-distributed.properties

步骤5.1：配置MySQL开启二进制日志并创建权限用户

my.ini

server-id         = 42
log_bin           = mysql-bin
binlog_format     = row
binlog_row_image  = full
expire_logs_days  = 10

DCL

MYSQL 受权kafka用户
//CREATE USER 'tandi'@'192.168.4.157' IDENTIFIED BY '123456';
//GRANT SELECT, RELOAD, SHOW DATABASES, REPLICATION SLAVE, REPLICATION CLIENT ON *.* TO //'tandi'@'192.168.4.157' IDENTIFIED BY '123456';
mysql> CREATE USER tandi IDENTIFIED BY '123456';
mysql> GRANT SELECT, RELOAD, SHOW DATABASES, REPLICATION SLAVE, REPLICATION CLIENT ON *.* TO tandi IDENTIFIED BY '123456';

步骤5.2：使用HTTP API让schema-registry server注册连接信息

POST请求 http://192.168.4.161:8083/connectors/
{
    "name": "my-first-connector",
    "config": {
        "connector.class": "io.debezium.connector.mysql.MySqlConnector",
        "database.hostname": "192.168.4.120",
        "database.port": "3306",
        "database.user": "tandi",
        "database.password": "123456",
        "database.server.id": "11111",
        "database.server.name": "192.168.4.120",
        "database.serverTimezone": "UTC",
        "database.history.kafka.bootstrap.servers": "192.168.4.161:9092",
        "database.history.kafka.topic": "192.168.4.120.db1" ,
        "include.schema.changes": "true",

        "transforms": "unwrap,changetopic",
        "transforms.unwrap.type": "io.debezium.transforms.UnwrapFromEnvelope",
        "transforms.changetopic.type":"org.apache.kafka.connect.transforms.RegexRouter",
        "transforms.changetopic.regex":"(.*)",
        "transforms.changetopic.replacement":"$1-smt"
    }
}

步骤6：启动消费者（注意顺序，第一次启动必须是先启动connect才启动消费者，因为connect成功才会生产对应的topic）

bin/kafka-avro-console-consumer --bootstrap-server 192.168.4.161:9092 --topic 192.168.4.120.db1.t1-smt --from-beginning

注意：如无意外，查看kafka topic会看到生成了一些以192.168.4.120开头的topic

步骤7：测试

配置方式2：仅使用Debezium

注意：配置好JDK环境下进行

server.properties（kafka配置文件)：修改或配置

listeners = PLAINTEXT://192.168.4.157:9092

connect-distributed.properties（kafka配置文件)

# 192.168.4.157 kafka服务器地址
bootstrap.servers=192.168.4.157:9092
# 将schemas.enable设置为false，或者在restful请求是显示设置schemas.enable为false，否者转换器不生效
key.converter.schemas.enable=false
value.converter.schemas.enable=false
# 配置RESTful服务
rest.host.name=192.168.4.157
rest.port=8083

将插件路径配置到环境变量中（无需在kafka的plugin.path中配置）

# user settings
JAVA_HOME=/usr/local/jdk1.8.0_201
PATH=$JAVA_HOME/bin:$JAVA_HOME/jre/bin:$NODE_HOME/bin:$PATH
export CLASSPATH=/usr/local/debezium-connector-mysql/*
export JAVA_HOME PATH

配置MySQL并创建权限用户（my.ini）

# 启动二进制日志（注意配置后重启MySQL服务）
server-id         = 11111
log_bin           = mysql-bin
binlog_format     = row
binlog_row_image  = full
expire_logs_days  = 10

在MySQL创建用户并授权（用于debezium读取mysql二进制文件）

mysql> CREATE USER tandi IDENTIFIED BY '123456';
mysql> GRANT SELECT, RELOAD, SHOW DATABASES, REPLICATION SLAVE, REPLICATION CLIENT ON *.* TO tandi IDENTIFIED BY '123456';

刪除：

  • delete http://192.168.4.157:8083/connectors/配置名称

  • get http://192.168.4.157:8083/connectors/配置名称

  • get http://192.168.4.157:8083/connectors/配置名称/status

  • get http://192.168.4.159:8083/connector-plugins
