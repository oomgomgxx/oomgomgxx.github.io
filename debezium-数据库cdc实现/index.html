使用Debezium实现数据库的CDC功能
发表于 2018-12-17| 更新于 2020-04-30|编程数据库
评论数:

是什么？

  • debezium是一个开源的分布式数据库事件流工具
  • debezium的功能类似于数据库的trigger，且比trigger功能强大
  • 可以用它来实现cdc功能（change data capture,CDC）

关于Debezium Connector for MySQL

  • debezium连接器通过观察mysql二进制日志的变动来生成操作事件，每个表中发生的所有事件数据都会“流动”到kafka对应的topic上

  • 连接器会在第一次连接mysql数据库的时候生成一个快照，完成后会从生成快照的那个点开始读取二进制日志，这样做的目的有两个：

    1）生成快照的原因：mysql配置（my.ini）存在过时清除设置，即可能会在某个时候二进制日志文件内容会被mysql清除

    2）从生成快照的那个点开始观察：从生成点读而不从生成后的点读原因是，预防在生成快照过程中数据库执行了修改操作

  • 连接器若中途故障，那么下次重启，会在故障点开始读取二进制日志；生成快照故障则是重新生成

下载Debezium和Confluent

Debezium：https://repo1.maven.org/maven2/io/debezium/debezium-connector-mysql/

Confluent：https://www.confluent.io/connector/debezium-mysql-cdc-connector/

简单说明：

  • debezium-connector-mysql：提供mysql二进制日志文件变动监听服务
  • debezium-mysql-cdc-connector：Confluent套件，提供CDC相应功能（自带kafka、zookeeper、avro等等功能），特别重要的RESTful server

配置方式1：使用Debezium + Confluent

注意：配置好JDK环境下进行

server.properties（kafka配置文件)：修改或配置

schema-registry.properties（confluent schema-registry组件配置文件)

connect-avro-distributed.properties（confluent connect avro格式连接配置文件)**

profile（linux环境变量）：注意插件变量名称一定要定义在CLASSPATH中，因为confluent提供的avro相关组件中的shell文件中使用到CLASSPATH

步骤4：启动kafka-connect

拓展：

kafka-connect RESTfult方式配置基本使用。相关文档如下

  • https://docs.confluent.io/current/connect/references/restapi.html
  • http://kafka.apache.org/documentation.html#connect

创建：

post http://192.168.4.157:8083/connectors/

刪除：

  • delete http://192.168.4.157:8083/connectors/配置名称

获取某配置的信息：

  • get http://192.168.4.157:8083/connectors/配置名称

获取某配置的状态：

  • get http://192.168.4.157:8083/connectors/配置名称/status

获取可用的插件信息（重要，可用于查看插件是否配置成功）：

  • get http://192.168.4.159:8083/connector-plugins
