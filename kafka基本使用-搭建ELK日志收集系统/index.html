搭建ELK日志收集系统

 2018-10-15   作者: TanDi   类型：原创

下载安装

kafka：http://kafka.apache.org/downloads

zookeeper：https://www.apache.org/dyn/closer.cgi/zookeeper/（不用kafka内置的zookeeper）

jdk：https://www.oracle.com/technetwork/java/javase/downloads/index.html

命令行操作

步骤1：配置jdk环境变量

1）vim /etc/profile

在unset...前添加一下内容
# user settings
JAVA_HOME=/usr/local/jdk1.8.0_201
PATH=$JAVA_HOME/bin:$JAVA_HOME/jre/bin:$PATH
export JAVA_HOME PATH

2）source /etc/profile 刷新配置

步骤2：将zookeeper配置文件zoo_sample.cfg改名为zoo.cfg

mv zoo_sample.cfg zoo.cfg

步骤3：启动zookeeper（使用kafka提供的配置文件）

[root@localhost bin]# ./zkServer.sh start /usr/local/kafka_2.11-2.1.0/config/zookeeper.properties

步骤4：启动kafka服务

[root@localhost bin]# ./kafka-server-start.sh ../config/server.properties

步骤5：创建topic

[root@localhost bin]# ./kafka-topics.sh --create --zookeeper 192.168.4.157:2181 --replication-factor 1 --partitions 1 --topic test

步骤6：查看topic是否创建成功

[root@localhost bin]# ./kafka-topics.sh --zookeeper 192.168.4.157:2181 --list
test

步骤7：kafka producer生产消息

[root@localhost bin]# ./kafka-console-producer.sh --broker-list 192.168.4.157:9092 --topic test

步骤8：kafka consumer消费消息

[root@localhost bin]# ./kafka-console-consumer.sh --bootstrap-server 192.168.4.157:9092 --topic test --from-beginning

–from-beginning ：表示监听 test这个topic上的所有历史消息（即新消息和旧消息都获取-> kafka消息持久化）

步骤4：代码

生产者

package com.td.resouce;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class KafkaResource {

    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    @GetMapping(value={"/","/login"})
    public String login(){
        return "ok";
    }

    @GetMapping("/send/{topic}/{content}")
    public String send(@PathVariable("topic") String topic,
                       @PathVariable("content") String content){
        try {
            System.out.println("准备发送消息...");
            kafkaTemplate.send(topic, content);
            System.out.println("准备发送完成...");
        }catch (Exception e){
            e.printStackTrace();
            return "send fail";
        }
        return "send success";
    }

}

消费者

package com.td.consumer;

import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Component;

@Component
public class KafkaConsumer {

    @KafkaListener(topics = "test")
    public void consumer(String content){
        System.out.println("监听到：" + content);
    }
}

Kafka Connect Elasticsearch Connector（方法1）

提醒：虚拟机复制的话切记要情况elasticsearch中的data目录数据

ElasticSearch直接消费kafka消息

以下以CDC为例

步骤1：下载连接器

kafka connect ElasticSearch：https://www.confluent.io/connector/kafka-connect-elasticsearch/

Debezium：https://repo1.maven.org/maven2/io/debezium/debezium-connector-mysql/

步骤2：安装并配置

profile（将Debezium插件依赖包配置到classpath上，必须）

# user settings
JAVA_HOME=/usr/local/jdk1.8.0_201
PATH=$JAVA_HOME/bin:$JAVA_HOME/jre/bin:$NODE_HOME/bin:$PATH
# debezium 插件依赖
export CLASSPATH=/usr/local/debezium-connector-mysql/*
export JAVA_HOME PATH

server.properties（kafka server配置）

listeners=PLAINTEXT://192.168.4.159:9092

connect-distributed.properties（kafka连接器配置）

bootstrap.servers=192.168.4.159:9092
key.converter.schemas.enable=false
value.converter.schemas.enable=false
# 启用rest配置schema
rest.host.name=192.168.4.159
rest.port=8083
# kafka connect elasticsearch插件
plugin.path=/usr/local/confluentinc-kafka-connect-elasticsearch-5.1.0

elasticsearch.yml

network.host: 192.168.4.159
http.port: 9200
# 允许跨域访问es（es插件）
http.cors.enabled: true
http.cors.allow-origin: "*"

步骤3：启动

  1. 启动zookeeper
  2. 启动kafka server
  3. 启动elasticsearch（注意：es必须使用非root启动）
  4. 启动kafka connect

Logstash整合kafka并数据到ElasticSearch（方法2）

Logstash消费kafka消息到ElasticSearch

步骤1：下载并上传Logstash到linux

logstash：https://www.elastic.co/cn/products/logstash

上传解压如下

步骤2：创建并配置Logstash启动文件

LogstashKafka.conf

[root@localhost config]# touch logstash-kafka.conf
[root@localhost config]# vim logstash-kafka.conf

配置文件说明：从input获取数据输出到output

从kafka读取 - > 从kafka中获取数据消费输出到控制台

output{  
    input{
      kafka{
        bootstrap_servers => ["192.168.4.157:9092"]
         # 建议和生产消息的线程组区别，同一个组不能消费同一个分区
        group_id => "consumer-logs-group"
        auto_offset_reset => "earliest"
        consumer_threads => 5
        decorate_events => true
        topics => ["logs","test"]
      }
    }

    stdout{
        codec => rubydebug
    }

    elasticsearch {
        hosts => ["192.168.4.157:9200"]
        user => "tandi"
        password => "tandi"
        index => "%{service}-%{+YYYY-MM-dd}"
    }
}

向kafka输出

input{
    stdin{}
}
output{
    kafka{
        topic_id => "logs"
        bootstrap_servers => "192.168.4.157:9092" 
        batch_size => 5

    stdout{
        codec => rubydebug
    }
}
