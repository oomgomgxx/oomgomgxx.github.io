从WordPress回到Hexo

 2018-05-23   作者: TanDi   类型：原创

前言

  用了WordPress一段时间，虽然插件和主题都非常丰富，但在编写文章方面略有些繁琐；因为我个人的习惯是先在本地使用Markdown编写好再复制到WordPress发布，但是这么一来，除了复制文字之外，还需要将图片一个个导进去多媒体库再修改文章中图片的地址，有些接受不了；而且对于低配的服务器来说，运行WordPress还是相较于静态的Hexo要吃力不少的；

使用阿里云服务器搭建hexo博客

使用 git + nginx 搭建hexo博客

安装git

yum install git

安装nginx并设置开机启动

yum install nginx
systemctl start nginx.service
systemctl enable nginx.service

创建提供hexo使用git库

为了简单我直接用了root来操作

1. 创建库

cd /usr/share/nginx && mkdir hexo.git && sudo chown -R root:root hexo.git && cd hexo.git && git init --bare

2. 配置git钩子

让其接收到push操作后重置hexo的静态页面

参考文章：http://wubaoguo.com/2016/12/02/版本控制/git钩子详解/

cd hooks && cp post-update.sample post-receive && vim post-receive

添加内容如下：

#!/bin/sh

# 删除之前的静态文件
cd /usr/share/nginx
rm -rf hexo

# 克隆最新的版本
git clone /usr/share/nginx/hexo.git

# 重启nginx
systemctl restart nginx.service

修改nginx映射路径

vim /etc/nginx/nginx.conf

配置如下

server {
  listen       80 default_server;
  listen       [::]:80 default_server;
  server_name  _;
  #root         /usr/share/nginx/html;
  root            /usr/share/nginx/hexo;  # 将80端口映射到hexo文件夹下

重新加载nginx配置

nginx -s reload

在本地安装hexo并提交到阿里云服务器

安装hexo需要node环境，node安装自行百度。

安装hexo

npm install hexo-cli -g
或者 cnpm install hexo-cli -g

创建本地hexo站点文件夹并初始化

mkdir hexo && cd hexo && hexo init -y .

s

配置hexo的_config.yml让其关联阿里云的git库

hexo文档：https://hexo.io/docs/deployment.html

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  # 格式：用户@服务器ip:服务器里git库的路径
  repo: root@127.0.0.1:/usr/share/nginx/hexo.git
  branch: master

安装hexo的git提交插件

npm install hexo-deployer-git --save # 发布到git库

将hexo发布到阿里云服务器中的git库

# 清除上一次hexo生成的静态文件
hexo clean
# 重新生成hexo的静态文件
hexo g
# 本地测试 npm install hexo-server --save
hexo s
# 将hexo静态文件发布到git库
hexo d

经过以上操作过后如意外的话，这是在浏览器直接输入服务的ip就可以访问到hexo了
