Nginx的基本使用

 2017-02-28   作者: TanDi

是什么

  • Nginx (engine x) 是一个高性能的HTTP和反向代理服务器。

优势

  • 速度快：

    比一般的web服务器响应速度快

  • 高扩展性：

    除了自带的模块还有支持第三方模块

  • 高可用性：

    设计优秀稳定，每个worker相对独立；master进程可以在worker进程出错时快速“拉起”新的worker子进程为其提供服务

  • 低内存消耗：

    一万个HTTP Keep-Alive连接在Nginx中只消耗2.5M内存

  • 热部署：

    支持不停止服务更新配置项、更新日志文等功能；可以在7*24小时（一周）不间断服务的前提下，升级Nginx的可执行文件；

常用来做什么

  • 反向代理
  • 负载均衡
  • http服务器（动静分离）
  • 正向代理

nginx -t   #测试配置是否有语法错误

nginx [-?hvVtq] [-s signal] [-c filename] [-p prefix] [-g directives]

-?,-h           : 打开帮助信息
-v              : 显示版本信息并退出
-V              : 显示版本和配置选项信息，然后退出
-t              : 检测配置文件是否有语法错误，然后退出
-q              : 在检测配置文件期间屏蔽非错误信息
-s signal       : 给一个 nginx 主进程发送信号：stop（停止）, quit（退出）, reopen（重启）, reload（重新加载配置文件）
-p prefix       : 设置前缀路径（默认是：/usr/local/Cellar/nginx/1.2.6/）
-c filename     : 设置配置文件（默认是：/usr/local/etc/nginx/nginx.conf）
-g directives   : 设置配置文件外的全局指令

基本配置

nginx.conf配置说明

# 设置nginx服务的系统使用用户
#user  nobody;

# 工作进程数量（一般和cpu个数一致就可以了）
worker_processes  1;

# 错误日志（不配置则启动一下的默认配置）
#error_log  logs/error.log;
#error_log  logs/error.log  notice;
#error_log  logs/error.log  info;

# nginx服务启动的pid
#pid        logs/nginx.pid;

# 事件
events {
    # 每个工作进程允许的最大连接数量（越多越好）
    worker_connections  1024;
}


# HTTP协议模块
# 关系：一个http下可以有多个server服务
# 说明：在http下的配置为全局配置，所有server服务配置可用；在server内的为局部配置；配置启动就近原则
http {

    # 将某些文件包含到配置中
    # 如下表示，将同目录下的 mime.types 配置文件包含进来
    include        mime.types;
    default_type  application/octet-stream;

    # 日志格式
    #log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
    #                  '$status $body_bytes_sent "$http_referer" '
    #                  '"$http_user_agent" "$http_x_forwarded_for"';

    # 访问日志
    access_log  logs/access.log  main;

    sendfile        on;
    #tcp_nopush     on;

    # 长连接，客户端和服务端的连接超时时间
    #keepalive_timeout  0;
    keepalive_timeout  65;

    #gzip  on;

    server {
        # 监听当前nginx所在机器的80端口
        listen       80;
        # 监听以localhost为域名访问当前nginx机器的请求 
        server_name  localhost;

        # 字符集
        #charset koi8-r;

        # 访问记录日志
        #access_log  logs/host.access.log  main;

        # 处理请求的资源
        # 例如：浏览器访问url"nginx机器ip:80"时，默认的处理页面
        # 相当于 GetMapping("/")
        location / {
            # 文件路径（html为nginx的根路径下的html文件夹）
            root   html;
            # 文件
            index  index.html index.htm;
        }


        # 当nginx返回一下http状态码时，对应错误码的处理页面
        #error_page  404              /404.html;
        error_page   500 502 503 504  /50x.html;

        location = /50x.html {
            # 文件路径
            root   html;
        }
    }

    # 将/etc/nginx/conf.d/*.conf配置导入（*.conf，为server配置；例如default.conf）
    # include /etc/nginx/conf.d/*.conf;
}

日志配置

nginx日志种类

  • access_log：访问日志，记录每次的请求
  • error.log：错误日志，记录错误的请求

日志配置

基本语法

配置日志语法（log_format）：log_format 格式变量名称 日志出入表达式

例子:

log_format  main  '$remote_addr - $remote_user [$time_local] "$http_host $request"'
                  '$status $body_bytes_sent "$http_referer" '
                  '"$http_user_agent" "$http_x_forwarded_for"';

可用于表达式的日志变量：

$args                    #请求中的参数值
$query_string            #同 $args
$arg_NAME                #GET请求中NAME的值
$is_args                 #如果请求中有参数，值为"?"，否则为空字符串
$uri                     #请求中的当前URI(不带请求参数，参数位于$args)，可以不同于浏览器传递的$request_uri的值，它可以通过内部重定向，或者使用index指令进行修改，$uri不包含主机名，如"/foo/bar.html"。
$document_uri            #同 $uri
$document_root           #当前请求的文档根目录或别名
$host                    #优先级：HTTP请求行的主机名>"HOST"请求头字段>符合请求的服务器名.请求中的主机头字段，如果请求中的主机头不可用，则为服务器处理请求的服务器名称
$hostname                #主机名
$https                   #如果开启了SSL安全模式，值为"on"，否则为空字符串。
$binary_remote_addr      #客户端地址的二进制形式，固定长度为4个字节
$body_bytes_sent         #传输给客户端的字节数，响应头不计算在内；这个变量和Apache的mod_log_config模块中的"%B"参数保持兼容
$bytes_sent              #传输给客户端的字节数
$connection              #TCP连接的序列号
$connection_requests     #TCP连接当前的请求数量
$content_length          #"Content-Length" 请求头字段
$content_type            #"Content-Type" 请求头字段
$cookie_name             #cookie名称
$limit_rate              #用于设置响应的速度限制
$msec                    #当前的Unix时间戳
$nginx_version           #nginx版本
$pid                     #工作进程的PID
$pipe                    #如果请求来自管道通信，值为"p"，否则为"."
$proxy_protocol_addr     #获取代理访问服务器的客户端地址，如果是直接访问，该值为空字符串
$realpath_root           #当前请求的文档根目录或别名的真实路径，会将所有符号连接转换为真实路径
$remote_addr             #客户端地址
$remote_port             #客户端端口
$remote_user             #用于HTTP基础认证服务的用户名
$request                 #代表客户端的请求地址
$request_body            #客户端的请求主体：此变量可在location中使用，将请求主体通过proxy_pass，fastcgi_pass，uwsgi_pass和scgi_pass传递给下一级的代理服务器
$request_body_file       #将客户端请求主体保存在临时文件中。文件处理结束后，此文件需删除。如果需要之一开启此功能，需要设置client_body_in_file_only。如果将次文件传 递给后端的代理服务器，需要禁用request body，即设置proxy_pass_request_body off，fastcgi_pass_request_body off，uwsgi_pass_request_body off，or scgi_pass_request_body off

$request_length          #请求的长度 (包括请求的地址，http请求头和请求主体)
$request_method          #HTTP请求方法，通常为"GET"或"POST"
$request_time            #处理客户端请求使用的时间,单位为秒，精度毫秒； 从读入客户端的第一个字节开始，直到把最后一个字符发送给客户端后进行日志写入为止。
$request_uri             #这个变量等于包含一些客户端请求参数的原始URI，它无法修改，请查看$uri更改或重写URI，不包含主机名，例如："/cnphp/test.php?arg=freemouse"

http_stub_status_module配置例子

作用：用于监控nginx客户端连接信息

1）配置 default.conf 文件

yum安装才有该文件(因为nginx.conf include了该文件)

如果源码安装，直接配置 nginx.conf 即可

server {
    listen       80;
    server_name  localhost;

    # configure http_stub_status_module
    location /mystatus {
        stub_status;
    }

    location / {
        root   /usr/share/nginx/html;
        index  index.html index.htm;
    }

    error_page   404 500 502 503 504  /50x.html;
    location = /50x.html {
        root   /usr/share/nginx/html;
    }
}

http_random_index_module配置例子

作用：

  1. 在root指定的目录下，随机选一个文件作为主页
  2. 注意：不会以.开头的文件作为主页（.开头为linux隐藏文件）

1）配置default.conf

server {
    listen       80;
    server_name  localhost;

    # configure http_stub_status_module
    location /mystatus {
        stub_status;
    }

    location / {
        root   /usr/share/nginx/html;

        #  configure http_random_index_module
        random_index on;
        # index  index.html index.htm;
    }

    error_page   404 500 502 503 504  /50x.html;
    location = /50x.html {
        root   /usr/share/nginx/html;
    }
}

3）浏览器访问

连接限制和请求限制

  1. http/https请求依赖于底层的tcp请求；tcp请求连接采用 3次 握手和 4次 挥手来处理连接和中断
  2. 其中一个连接可以有多个请求（http1.1）
  3. 一般用于防止恶意请求服务器导致服务器瘫痪或影响性能

配置语法：

limit_conn_zone

语法: limit_conn_zone $variable zone=name:size;
作用：限制连接数
默认值: none
配置段: http

例子：
limit_conn_zone $binary_remote_addr zone=addr:1m;
说明》》
当$variable为$binary_remote_addr时，表示以客户端ip作为限制依据
addr:1m：表示生成一个大小为1M，名字为addr的内存区域，用来存储访问的频次信息
表示:区域名称为addr（自定义），占用空间大小为1m

limit_req_zone

语法: limit_req_zone $variable zone=name:size rate=rate;
作用：限制请求频率
默认值: none
配置段: http

例子：
limit_req_zone $binary_remote_addr zone=req_zone:2m rate=1r/s;
说明》》
当$variable为$binary_remote_addr时，表示以客户端ip作为限制依据
req_zone:2m：表示生成一个大小为2M，名字为req_zone的内存区域，用来存储访问的频次信息
rate=1r/s：表示允许相同标识的客户端的访问频次，这里限制每秒1次。还可以有比如30r/m的，即限制每2秒访问一次，即每2秒才处理一个请求
表示:区域名称为req_zone（自定义），占用空间大小为2m，平均处理的请求频率不能超过每秒1次

测试例子

1）default.conf配置

# 定义限制条件策略
limit_conn_zone $binary_remote_addr zone=conn_zone:1m;
limit_req_zone $binary_remote_addr zone=req_zone:500m rate=1r/s;

server {
    listen       80;
    server_name  localhost;

    location / {
        root   /usr/share/nginx/html;

        # 表示同一时刻，只允一个ip连接
        #limit_conn conn_zone 1;
        # burst=3：表示在rate=1r/s基础上再延时处理3个请求
        #limit_req zone=req_zone burst=3;
        # burst=3 nodelay：表示在rate=1r/s基础上再不延时地处理3个请求
        #limit_req zone=req_zone burst=3 nodelay;
        # 频率处理限制为1个，同一时间内只处理一个请求
        limit_req zone=req_zone;

        index  index.html index.htm;
    }

    error_page   404 500 502 503 504  /50x.html;
    location = /50x.html {
        root   /usr/share/nginx/html;
    }
}

syntax: deny [ address | CIDR | all ]
default: no
context: http, server, location, limit_except # 可配置的位置

使用例子：

静态资源服务

什么是静态资源？

服务器直接能返回无需处理的资源，如图片、视频、文件、html、css、js等等

提高静态资源加载效率的方案

  • 浏览器缓存
  • CDN静态资源缓存（nginx作静态资源的分发；分发到client端经过的（距离近）的CDN保存；使用sendfile配置模块）

配置

sendfile配置

作用：用于读取静态资源文件

syntax: sendfile [ on|off ]
default: sendfile off
context: http, server, location

tcp_nopush配置

作用：在sendfile开启情况下，提高网络包的传输效率；一般用于大文件

syntax: tcp_nopush [on|off]
default: tcp_nopush off
context: http, server, location

gzip相关配置

【gzip】
语法: gzip on|off
默认值: gzip off
作用域: http, server, location, if (x) location
作用：用于传输时文件压缩；例如浏览器访问静态资源时获取的静态文件会较小，所以速度就加快

【gzip_comp_level】
语法: gzip_comp_level 1..9
默认值: gzip_comp_level 1
作用域: http, server, location
作用：gzip压缩比，1 压缩比最小处理速度最快，9 压缩比最大但处理最慢（传输快但比较消耗cpu）

【gzip_types】
语法: gzip_types mime-type [mime-type ...]
默认值: gzip_types text/html
作用域: http, server, location
作用：匹配MIME类型进行压缩，（无论是否指定）"text/html"类型总是会被压缩的

正向代理

例子1-vpn：

vpn就是正向代理，国内无法访问google，这时需要使用一台能够访问google的机器作为代理服务器，那么我们的机器连接上该机器即可达到访问google的目的

有一些公司只允许使用内容，目的是提高安全性，而整个公司可能只有一台作为关口可以连接网络的服务器，这只需要在浏览器将这台服务器配置成代理才能够上网

反向代理

说明：

  1. 反向代理和正向代理相反，反向代理代理的是”内网”中的服务器，即代理的对象不同，这里的“内网“指的是不直接对外提供服务的服务器；

  2. 而反向代理服务器，就是作为这些（1-多台）”内网”服务器的关口服务器，来代表他们对外提供服务，但需要清楚的是处理请求的还是”内网“中的服务器；

  3. 常见的反向代理就是，有多台提供服务的服务器（对用户透明），这时使用一个反向代理服务器来代理这多台提供服务的服务器对外提供服务；如淘宝这些大型购物网站
