Nginx使用

基本介绍

是一个基于多进程模型及事件驱动的轻量服务器

特性

  • 高并发、高性能、高可用
    • Nginx 会确保每一个连接消耗的内存最小化
    • Master 进程可以在 Worker 进程出错后快速拉起新的Worker 进程继续提供服务
    • 使用 Reactor 模型处理网络请求（在 Nginx 中准确来说应该叫 Run-Loop）
  • 可扩展性高（模块化设计），第三方模块丰富
  • 热部署
    • 支持不停机(信号)维护进程，以及更新配置项和日志文件
  • 源码使用BSD许可证，给予使用者非常高的使用自由度

常见使用场景

  • 静态资源服务器
    • 当前文件系统作为静态资源存储点提供服务
  • 反向代理服务器
    • 基于多进程事件驱动性能强大
    • 提供静态资源缓存功能，能一定程度提高响应速度
    • 提供负载均衡能力，可合理地分摊上游服务器负载
  • 微服务服务API网关
    • Nginx + Lua
  • 流媒体服务器
    • 可通过 nginx-rtmp 模块实现

编译安装

1）在官网选择需要的版本

  • https://nginx.org/en/download.html

2）下载后解压

> wget https://nginx.org/download/nginx-1.14.2.tar.gz
> tar -zxvf nginx-1.14.2.tar.gz

3）生成编译文件 Makefile

> cd nginx-1.14.2
> ./configure --prefix=/opt/nginx
  • 可以通过./configure --help查看配置参数
  • –prefix 用于指定安装路径
  • 如果需要安装指定模块，可以参考 –with- 开头的参数；反之参考 –without- 开头的参数
  • 执行完./configure后会在当前目录生成一个objs文件夹，其中 ngx_modules.c 中包含将要安装的模块

3）安装依赖

> yum install -y gcc gcc-c++ autoconf pcre pcre-devel make automake httpd-tools zlib zlib-devel openssl openssl-devel

4）更具 Makefile 编译和安装

> make && make install

5）将Nginx的命令添加到环境变量（可选操作）

  • 将 sbin 目录配置到 /etc/profile 中的 PATH 环境变量尾部
  • 修改完执行source /etc/profile刷新配置即可

6）使用 vim 编辑 Nginx 配置文件时高亮显示（可选操作）

> pwd
/root/nginx-1.14.2
> mkdir ~/.vim
> cp -r contrib/vim/* ~/.vim

命令使用

nginx命令

> nginx -h
nginx [-?hvVtq] [-s signal] [-c filename] [-p prefix] [-g directives]

-?,-h           : 打印帮助信息
-v              : 打印显示版本信息
-V              : 打印显示版本信息，以及 configure 配置项
-t              : 测试配置文件
-q              : 在检测配置文件期间屏蔽非错误信息
-s signal       : 给Master进程发送信号：stop（停止）, quit（退出）, reopen（开启新日志）, reload（重新加载配置）
-p prefix       : 指定nginx目录
-c filename     : 指定配置文件
-g directives   : 指定附加配置路径

kill命令

在 Nginx 中除了可以使用 nginx -s 给进程发送信号外，其实还可以通过命令 kill 发送

1）支持的操作信号：

> kill -l
HUP INT QUIT ILL TRAP ABRT BUS FPE KILL USR1 SEGV USR2 PIPE ALRM TERM STKFLT CHLD CONT STOP TSTP TTIN TTOU URG XCPU XFSZ VTALRM PROF WINCH POLL PWR SYS
  • 注意以上信号都有与之对应的编号，从1开始往右递增。

2）常用信号：

编号 信号 功能
1 HUP 平滑重启。用新配置启动新进程，然后关闭旧进程
2 INT 中断信号
3 QUIT 平滑退出信号
9 KILL 强制杀死进程
15 TERM （默认）终止信号。和 INT 一样都数据快速停止信号
18 CONT 从 STOP 恢复运行信号
19 STOP 暂停信号
10 USR1 开启新日志。常用来做日志切割
12 USR2 平滑升级。使用新版本启动进程，然后关闭旧版本进程
28 WINCH 平滑地关闭 Worker 进程。常和USR2一同使用于平滑升级

案例：为正在运行的 Nginx 添加新模块

1）下载第三方模块

> git clone https://github.com/replay/ngx_http_consistent_hash.git

2）进入源文件夹重新生成 Makefile

> nginx -V # 查看之前的配置项，做到在原来的基础上添加
nginx version: nginx/1.14.2
built by gcc 8.3.1 20191121 (Red Hat 8.3.1-5) (GCC) 
configure arguments: --prefix=/opt/nginx
> ./configure --prefix=/opt/nginx --add-module=/root/ngx_http_consistent_hash

3）重新根据 Makefile 编译

> make
  • 注意无需执行make install 重新安装，否则现有的 Nginx 目录会被覆盖

4）将 objs 中编译好的二进制 Nginx 命令拷贝到安装目录

# 先备份预防万一
> mv /opt/nginx/sbin/nginx /opt/nginx/sbin/nginx.bak
> cp objs/nginx /opt/nginx/sbin

5）验证

> nginx -V
nginx version: nginx/1.14.2
built by gcc 8.3.1 20191121 (Red Hat 8.3.1-5) (GCC) 
configure arguments: --prefix=/opt/nginx --add-module=/root/ngx_http_consistent_hash # 可以看到新模块已经加入

6）平滑升级

> kill -USR2 14180 # 平滑升级，这里会使用新的二进制文件启动Nginx
> kill -WINCH 14180 # 通知旧Master进程让其关闭Worker进程
> kill -QUIT 14180 # 平滑地关闭旧Master进程

模块配置了解

全局块（对整个Nginx起作用）

  • 指定 Nginx 的管理用户及其组信息
  • 指定启动时拉起的 Worker 进程的数量
  • 进程 PID 号的存放位置
  • 日志存放路径和日志类型配置
  • 引入其它配置

events块

  • 指定 Worker 进程同一时刻能够连接客户端的最大数量
    • 最大连接数 = worker_processes * worker_connections
    • 注意：worker_connections 不能超过文件系统所允许打开的最大文件数量。可以通过 cat /proc/sys/fs/file-max 查看
  • 事件驱动模型选择
  • 是否允许同时接受多个网络连接

http模块

  • 媒体类型
  • 代理配置
  • 缓存配置
  • 日志定义
  • 连接配置
  • 压缩配置
  • 零拷贝配置

server块

  • 虚拟主机。可配置多个
  • 监听端口
  • 域名关联
  • 错误码页面配置
  • 安全通信配置

location块

  • 虚拟主机资源。可配置多个
