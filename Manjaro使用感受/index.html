Manjaro Linux

 2019-09-04   作者: TanDi

闲聊

  先前一直使用WIN10作为主系统，但因为在WIN10中使用Docker时需要开启Hyper-V功能，而且开启Hyper-V之后无法使用64bit的虚拟机系统，因此才考虑到更换系统;

  在这之前也使用过诸如CentOS、Ubuntu、RedHat、Mint等Linux的发行版，因为考虑到工作和日常使用兼顾，所以原本首先考虑的是Ubuntu和Mint;

  据我自己了解Ubuntu和Debian这两个发行版在国内的使用和社区都是比较活跃的，其实不论是Linux发行版也好还是其它的开源技术也好，用户人数、社区的活跃程度还是版本的迭代速度都可以正面地体现产品的稳定性，因为多人用和社区活跃的话，发现问题和修复问题的速度就相对较快因此而保证了产品的质量;

  其中Ubuntu在大学时使用过，用的是gnome版本，界面不错但是使用感受并不是很好，给我的感觉是软件总是报些莫名其妙的错误，如系统崩溃…，当然可能是我自己的使用问题，它的优点是安装软件较为方便，而且国内的某些软件也支持Ubuntu和Debian等版本而无需过多的折腾，总体来说能够满足日常的使用;

  至于Mint是我在youtube上看大神写代码时了解到的，界面也相当简洁和漂亮，但不管是Ubuntu还是Mint，都是基于Debian的，个人更加偏向于Mint，因为不太想折腾搞些花里胡哨的东西;

  好了说到Manjaro，Manjaro是Arch Linux的衍生版，虽然我并没有用过Arch，但是看到Manjaro近来的形势相当不错，如下载排名、评价都十分优异，于是我首先在虚拟机中安装进行了尝试，直观的感受是简洁、方便、软件丰富（这是重点），而从近段时间作为主系统来使用的表现也相当稳定，所以个人非常推荐

制作系统盘安装Manjaro

步骤1：下载Manjaro镜像

https://manjaro.org/download/

需要说的是，Manjaro提供的丰富的界面版本（如KDE、XFCE、GNOME），根据个人喜好选择下载即可

步骤2：下载Rufus用于制作系统盘

http://rufus.ie/

据目前的了解Rufus只支持windows，既需要在Windows上制作;

Rufus相对于老毛桃、大白菜、UltraISO这些软件更加方便和简洁

建议

  1. 安装参考：https://linuxhint.com/install_manjaro_linux/

  2. 建议联网安装

界面展示

软件方面比较丰富，基本日常用的都有，而且一些windows软件可以直接通过arc构造十分方便

初始化操作配置

更新系统

sudo pacman -Syyu

设置国内软件源

sudo pacman-mirrors -i -c China -m rank     //选择中国源并更新
sudo pacman -S archlinuxcn-keyring

添加Archlinuxcn软件源

sudo vim /etc/pacman.conf，追加一下内容

[archlinuxcn]
SigLevel = Optional TrustedOnly
Server = https://mirrors.tuna.tsinghua.edu.cn/archlinuxcn/$arch

将home中的中文目录改为英文

sudo pacman -S xdg-user-dirs-gtk
export LANG=en_US
xdg-user-dirs-gtk-update
export LANG=zh_CN.UTF-8

更改登录背景壁纸

curl -L -O http://archibold.io/sh/archibold
chmod +x archibold
sudo mv archibold /usr/bin
sudo archibold login-background [壁纸的位置]

安装命令工具

支持ifconfig命令

sudo pacman -S net-tools

支持dig、nslookup命令

sudo pacman -S dnsutils

支持iostat、iotop

sudo pacman -S sysstat
sudo pacman -S iotop

安装搜狗拼音

sudo pacman -S fcitx-im
sudo pacman -S fcitx-configtool
sudo pacman -S fcitx-sogoupinyin
sudo pacman -S yaourt
yaourt -S fcitx-qt4

推荐软件

Plank：软件栏

Albert：搜索工具，功能如Mac Spotlight

BleanchBit：垃圾缓存清理工具

Drawing：画图工具，如Windows上的画板

Peek：GIF截图工具

深度截图：代替snipaste截图工具

深度终端：比自带的terminal功能丰富

Dconf：系统配置文件编辑器

Shadowsocks-Qt5：小飞机客户端

Conky：硬件指标监控工具

GoldenDict：代替StarDict的词典

MPV：视频播放器

配置非全局网络代理

genpac --pac-proxy "SOCKS5 127.0.0.1:1080" --gfwlist-proxy="SOCKS5 127.0.0.1:1080" --gfwlist-url=https://raw.githubusercontent.com/gfwlist/gfwlist/master/gfwlist.txt --output="autoproxy.pac" 

genpac --proxy="SOCKS5 127.0.0.1:1080" -o autoproxy.pac

问题

解决百度云盘无法登录问题

方法1：删除当前用户的baidu文件夹

rm -rf .local/share/baidu

方法1拓展：方法1重启失效，既每次重启完运行都需要再次删除，那么可以将这个操作设置成开机执行

sudo pacman -Sy systemd-rc-local # 因为manjaro基于arch，而arch是不支持rc.local的
sudo systemctl enable rc-local.service
sudo touch /etc/rc.local
sudo chmod 755 /etc/rc.local
sudo vim /etc/rc.local
添加如下内容
#!/bin/bash
rm -rf /home/用户名称/.local/share/baidu

方法2：使用root运行程序

解决网易云音乐无法输入中文问题

源自网友kmpsliu

1.先安装qcef这个软件包

sudo pacman -S qcef

2.编辑/opt/netease/netease-cloud-music/netease-cloud-music.bash，把它改成这样：
#!/bin/sh

HERE="$(dirname "$(readlink -f "${0}")")"
export LD_LIBRARY_PATH=/usr/lib
export QT_PLUGIN_PATH="${HERE}"/plugins
export QT_QPA_PLATFORM_PLUGIN_PATH="${HERE}"/plugins/platforms
exec "${HERE}"/netease-cloud-music $@

3.安装vlc这个包

sudo pacman -S vlc

解决wine中的软件字体小

env WINEPREFIX="$HOME/.deepinwine/应用" winecfg

安装驱动导致无法进入系统界面（Started TLP system startup/shutdown）

查看驱动

mhwd -li

删除驱动

mhwd -f -r 驱动类型 驱动名字
例子：mhwd -f -r pci video-vmware

软件降级

sudo pacman -S downgrade
sudo downgrade 软件名称

忽略软件更新

在/etc/pacman.conf中添加忽略配置

IgnorePkg=软件名称

或者在软件管理中设置
