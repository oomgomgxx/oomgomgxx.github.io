内容修订

基本介绍

  • Spring Security 用于解决 Web 应用的安全访问问题（譬如对用户进行认证和鉴权等操作）
  • Spring Security 基于 Spring AOP 和 Filter 实现。提供了一系列的 Filter 来解决 Web 应用上的安全问题
  • 类似于 Spring Security 的安全框架还有比较著名的 Apache Shiro，但因为 Spring Security 是 Spring 全家桶的成员，因此能够与 Spring Framework 无缝集成，而且 Spring Security 对目前流程的 Oauth2 开放授权协议有着良好的支持，所以对于目前而言学习 Spring Security 是很有必要的

关于认证、授权、鉴权

  • 认证：通过凭证来证明是当前系统的合法用户
  • 授权：在用户通过认证后，系统给予用户资源权限的过程
  • 鉴权：在系统给用户授予（特定）权限之后，用户再访问资源时系统就需要对其当前的访问操作进行资格判断。即有权限可操作性，无权限则拦截

SecurityConfig

@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Autowired
    private UserRepository userRepository;
    @Autowired
    private RoleRepository roleRepository;

    @Override
    public UserDetailsService userDetailsService() {
        return new UserDetailsService() {
            // 从数据库中返回用户信息
            @Override
            public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
                User user = userRepository.findUserByUserName(username);
                List<Role> roles = roleRepository.findRolesByUserId(user.getuId());
                return new UserPrincipal(user, roles);
            }
        };
    }


    @Bean
    public DaoAuthenticationProvider authenticationProvider() {

        // 密码加密工具类
        Map<String, PasswordEncoder> encoders = new HashMap<>();
        encoders.put("pbkdf2", new Pbkdf2PasswordEncoder());

        DaoAuthenticationProvider authenticationProvider = new DaoAuthenticationProvider();
        authenticationProvider.setUserDetailsService(userDetailsService());
        authenticationProvider.setPasswordEncoder(new DelegatingPasswordEncoder("pbkdf2", encoders));
        return authenticationProvider;
    }

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        // 要求用户访问某个uri时，需要有指定的角色
        http.authorizeRequests()
                .antMatchers("/user/**").access("hasRole('ROLE_ADMIN') or hasRole('ROLE_USER') or hasRole('ROLE_DBA')")
                .antMatchers("/admin/**").access("hasRole('ROLE_ADMIN') ")
                .antMatchers("/dba/**").access("hasRole('ROLE_ADMIN') or hasRole('ROLE_DBA')")
                .and().formLogin();
    }

}

UserPrincipal

public class UserPrincipal implements UserDetails {
    // 也可以直接通过User来实现UserDetails
    private User user;
    private List<Role> roles;

    public UserPrincipal(User user, List<Role> roles) {
        this.user = user;
        this.roles = roles;
    }

    // 权限
    @Override
    public Collection<? extends GrantedAuthority> getAuthorities() {
        return roles.stream().map(role -> new SimpleGrantedAuthority(role.getrName())).collect(Collectors.toList());
    }

    @Override
    public String getPassword() {
        return user.getPassWord();
    }

    @Override
    public String getUsername() {
        return user.getUserName();
    }


    @Override
    public boolean isAccountNonExpired() {
        return true;
    }

    @Override
    public boolean isAccountNonLocked() {
        return true;
    }

    @Override
    public boolean isCredentialsNonExpired() {
        return true;
    }

    @Override
    public boolean isEnabled() {
        return true;
    }

    public User getUser() {
        return user;
    }

    public void setUser(User user) {
        this.user = user;
    }

    public List<Role> getRoles() {
        return roles;
    }

        this.roles = roles;

关于 HttpSecurity 常用配置

HttpSecurity 是 Spring Security 对请求资源路径进行拦截的配置

@Configuration
@EnableWebSecurity
public class CustomWebSecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
                .antMatchers("/resources/**", "/about").permitAll()
                .antMatchers("/admin/**").hasRole("ADMIN")
                .antMatchers("/db/**").access("hasRole('ADMIN') and hasRole('DBA')")
                .anyRequest().authenticated()
                .and()
            .formLogin()
                .usernameParameter("username")
                .passwordParameter("password")
                .failureForwardUrl("/login?error")
                .loginPage("/login")
                .permitAll()
                .and()
            .logout()
                .logoutUrl("/logout")
                .logoutSuccessUrl("/index")
                .permitAll()
                .and()
            .httpBasic()
                .disable();
    }
}
  • authorizeRequests()：授权请求定义
  • permitAll()：表示无需授权，允许直接访问
  • antMatchers()：表示对应的请求路径uri
  • anyRequest()：表示其余的所有请求
  • authenticated()：表示需要认证
  • and()：代表一段定义的结束并连接下一段其他的定义
  • formLogin()：登录认证定义
  • loginPage()：指定登录路径
  • usernameParameter()：指定登录认证表单提交的username字段名称
  • passwordParameter()：指定登录认证表单提交的password字段名称
  • failureForwardUrl()：指定登录认证失败路径
  • logout()：登出认证定义
  • logoutUrl()：指定登出路径
  • httpBasic()：basic登录定义
  • disable()：表示不可用

LoginSuccessHandler

public class LoginSuccessHandler implements AuthenticationSuccessHandler {

    @Override
    public void onAuthenticationSuccess(HttpServletRequest request, HttpServletResponse response, Authentication authentication) throws IOException, ServletException {

        // SavedRequest缓存了当前的request请求
        SavedRequest savedRequest = new HttpSessionRequestCache().getRequest(request, response);

        // 获取到原来请求的url
        // 例如在认证前访问的是localhost:8080/admin,那么认证成功后就重定向到admin
        String redirectUrl = (savedRequest == null) ? "" : savedRequest.getRedirectUrl();
        if (!redirectUrl.isEmpty()) {
            response.sendRedirect(redirectUrl);
            return;
        }

        // 如果为null，则调到所属页面
        // 获取认证成功的用户信息
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();

        for (GrantedAuthority authority : auth.getAuthorities()) {

            // 用户的权限
            String role = authority.getAuthority();

            if (role != null && !"".equals(role)) {
                switch (role) {
                    case "ROLE_ADMIN":
                        redirectUrl = "/admin";
                        break;
                    case "ROLE_DBA":
                        redirectUrl = "/dba";
                        break;
                    default:
                        redirectUrl = "/user";
                        break;
                }
            }
        }

        // 重定向到对应的页面
        response.sendRedirect(redirectUrl);
    }
}

认证的基本流程

自定义 AuthenticationProvider (认证条件)

MyProvider

@Component
public class MyProvider implements AuthenticationProvider {

    private UserDetailsService userDetailsService;
    public MyProvider(UserDetailsService userDetailsService) {
        this.userDetailsService = userDetailsService;
    }

    // 我们的自定义认证条件是，只要用户名正确就可以了不需要校验密码
    @Override
    public Authentication authenticate(Authentication authentication) throws AuthenticationException {

        // 获取表单提交的用户名
        String username = authentication.getName();

        // 根据提交的用户名到数据库中查找
        UserDetails userDetails = userDetailsService.loadUserByUsername(username);
        Authentication result = authentication;

        // 如果相等则直接返回
        if (userDetails != null) {
            // 相当于凭证已认证的Authentication
            result = new UsernamePasswordAuthenticationToken(
                    userDetails.getUsername(),
                    userDetails.getPassword(),
                    userDetails.getAuthorities());
        }

        return result;
    }

    @Override
    public boolean supports(Class<?> authentication) {
        return true;
    }
}

JwtRequestFilter

@Component
public class JwtRequestFilter extends OncePerRequestFilter {
    @Autowired
    private UserDetailsService userDetailsService;
    @Autowired
    private JwtUtil jwtUtil;
    @Autowired
    private ObjectMapper objectMapper;

    @Override
    protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response, FilterChain filterChain)
            throws ServletException, IOException {

        // 从请求中获取token
        String token = request.getHeader(SecurityConst.AUTHENTICATE_HEAD);
        if (token == null)
            token = request.getHeader("token") == null ? request.getParameter("token") : null;

        // 校验token
        if (!StringUtils.isEmpty(token)) {
            String tokenUsername = null;

            if (!jwtUtil.validateToken(token))
                authFailResponse(response, "token已过期", 401);

            try {
                tokenUsername = jwtUtil.extractUsername(token);
            } catch (ExpiredJwtException e) {
                authFailResponse(response, "token解析失败", 401);
                return;
            }

            if (tokenUsername != null) {

                // 从数据库获取UserDetails
                UserDetails userDetails 
                    = userDetailsService.loadUserByUsername(tokenUsername);

                // 生成未认证token
                // 这里之所以直接将数据库获得的userDetails传入，是因为token有效
                // 因此无需再认证
                UsernamePasswordAuthenticationToken authenticationToken
                        = new UsernamePasswordAuthenticationToken(userDetails, null, userDetails.getAuthorities());

                try {
                    // 如果是一般登录，则需要通过以下方法认证进行认证
                    // Authentication authentication = authenticationManager.authenticate(authenticationToken);

                    // 保存到SecurityContext中，提供后续鉴权使用
                    SecurityContextHolder.getContext().setAuthentication(authentication);

                } catch (AuthenticationException e) {
                    authFailResponse(response, "认证失败", 401);
                    return;
                }
            }
        }

        // 继续执行拦截器链
        filterChain.doFilter(request, response);
    }

    private void authFailResponse(HttpServletResponse response, String respMsg, int statusCode) {
        try {
            response.setContentType("application/json; charset=utf-8");
            response.setStatus(statusCode);
            JsonData jsonData = JsonData.buildError(respMsg);
            String resp = objectMapper.writeValueAsString(jsonData);
            response.getWriter().println(resp);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

将 JwtRequestFilter 添加到过滤器链中

@EnableWebSecurity
@EnableGlobalMethodSecurity(prePostEnabled = true)
public class SecurityConfig extends WebSecurityConfigurerAdapter {    
    //...

    @Override
    protected void configure(HttpSecurity httpSecurity) throws Exception {
        // 端点连接配置
        httpSecurity
                .csrf().disable()
                .formLogin().disable()
                .anonymous().disable()
                .httpBasic().disable() 
                .logout().disable()
                .cors()
                .and()
                .authorizeRequests()
                .anyRequest().authenticated() // 所以请求都需要认证
                .and()
            .sessionManagement()
            .sessionCreationPolicy(SessionCreationPolicy.STATELESS);

        // 添加JWT Token拦截过滤器
        httpSecurity
           .addFilterBefore(jwtRequestFilter,UsernamePasswordAuthenticationFilter.class);
    }

    @Override
    public void configure(WebSecurity web) throws Exception {
        // 排除不需要拦截的资源路径
        // 譬如：登录、swagger、静态资源
        String[] uris = excludedProperties.getExcludedURIsToArray();
        web.ignoring().antMatchers(uris);
    }
}

一些核心类和接口介绍

Authentication

Authentication是一个接口，譬如 *Token 结尾的类都实现自该接口。

用于存储用户的认证信息。譬如常见的 UsernamePasswordAuthenticationToken

SecurityContextHolder

SecurityContextHolder 是一个获取 SecurityContext 的静态工具类，而 SecurityContext 为当前请求的认证授权上下文。

// 通过工具获取上下文对象
SecurityContext context = SecurityContextHolder.getContext();
// 当前上下文的保存策略
SecurityContextHolder.getContextHolderStrategy();
// 获取认证信息
Authentication authentication 
    = context.getAuthentication();

// Authorities用户拥有的权限
authentication.getAuthorities();
// Credentials用户凭证，一般为密码
authentication.getCredentials();
// Principal用户身份（标识属性），一般为用户名、Email、手机号、身份证等信息
authentication.getPrincipal();
// Details当前认证用户的额外信息
authentication.getDetails();

AuthenticationManager和AuthenticationProvider

  • AuthenticationManager 本身是一个接口，它只有一个抽象方法 Authentication authenticate(authentication)，用于用户认证操作（传入需要认证的authentication，成功则返回已认证的authentication）。
  • 其中 AuthenticationManager 有个实现叫 ProviderManager，它用于管理 AuthenticationProvider（认证规则） 集合
  • 在认证过程中，需要认证的 authentication 会经过 AuthenticationProvider 集合，而重要的是只要有一个 AuthenticationProvider 返回 Authentication 就表示认证认证通过
  • 因此，如果需要实现自定义认证规则，只需要实现一个 AuthenticationProvider 并添加到 AuthenticationManager 即可

UserDetailsService

UserDetailsService 是一个接口，其实现用通过用户提交的 Principal 来查询 UserDetails。

PasswordEncoder

PasswordEncoder是一个接口，其实现用于给用户密码进行加密。

HttpSecurity

HttpSecurity 是一个 immutable 类，用于设置 HTTP 请求拦截规则。

AuthenticationManagerBuilder

使用 AuthenticationManagerBuilder 可以构建出基于特定环境和边界的AuthenticationManager。

    @Autowired
    public void configureGlobal(AuthenticationManagerBuilder auth) throws Exception {
        auth.inMemoryAuthentication().withUser("tandi01").password("{noop}123456").roles("USER");
        auth.inMemoryAuthentication().withUser("tandi02").password("{noop}123456").roles("ADMIN");
        auth.inMemoryAuthentication().withUser("tandi03").password("{noop}123456").roles("DBA");
    }
