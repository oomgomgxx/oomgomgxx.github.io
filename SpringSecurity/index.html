基本介绍

  • spring security用于解决web应用的安全问题；如对用户的认证，判断用户是否为非法用户；对用户的授权，指定哪些用户能够访问和操作哪些功能；
  • 类似于spring security的安全框架还有apache的shiro，但spring security是spring全家桶的一员，能够与spring framework无缝集成，而且spring security对Oauth2认证授权标准有良好的支持，所以学习spring security对于目前的需求来说是有必要的；
  • spring security是一种基于 Spring AOP 和 Servlet 过滤器的安全框架；
  • spring security提供了一系列的Filter来解决web应用的安全问题

模拟场景

  • web应用中存在3个需要认证和授权的页面，分别是：user.jsp、admin.jsp、dba.jsp
  • web应用中存在3个角色，分别是：USER、ADMIN、DBA
  • 访问页面需要对应的权限，关系如下：

    • admin.jsp 要求用户必须有 ADMIN 角色才可以访问
    • dba.jsp 要求用户拥有 ADMIN、DBA 中其中一个角色就可以访问

关于认证和授权

  • 所谓的认证，通俗点来说就是证明身份的过程；即验证你就是系统中的某个用户
  • 授权，授权是系统对用户而言的，即系统给用户授予某些功能的使用权力

关于HttpSecurity常用配置

HttpSecurity配置是spring security对请求的拦截配置

  • authorizeRequests()：授权请求定义
  • permitAll()：表示无需授权，允许直接访问
  • antMatchers()：表示对应的请求路径uri
  • anyRequest()：表示其余的所有请求
  • authenticated()：表示需要认证
  • and()：代表一段定义的结束并连接下一段其他的定义
  • formLogin()：登录认证定义
  • loginPage()：指定登录路径
  • usernameParameter()：指定登录认证表单提交的username字段名称
  • passwordParameter()：指定登录认证表单提交的password字段名称
  • failureForwardUrl()：指定登录认证失败路径
  • logout()：登出认证定义
  • logoutUrl()：指定登出路径
  • httpBasic()：basic登录定义
  • disable()：表示不可用

spring security中一些核心类和接口

Authentication

Authentication是一个接口，*Token都实现了该接口，用于存储用户的认证信息（未认证和已认证都用Authentication存放）常见的UsernamePasswordAuthenticationToken就是它的实现

SecurityContextHolder

SecurityContextHolder是一个获取SecurityContext的静态工具类，而SecurityContext可以获取到当前请求的认证信息。可以利用SecurityContextHolder获取如下信息：

AuthenticationManager和AuthenticationProvider

从名字就可以看出来意思，AuthenticationManager用于管理AuthenticationProvider；

其中AuthenticationManager是一个接口，默认的实现叫ProviderManager，而ProviderManager中有个叫parent的属性用于指向自己的父ProviderManager，形成链表，所以得知AuthenticationProvider是可以有多个，其中只要有一个返回Authentication就表示认证成功；

UserDetailsService

UserDetailsService是一个接口，提供使用用户名称查找返回UserDetails服务

PasswordEncoder

PasswordEncoder是一个接口，功能是给用户密码编码加密

HttpSecurity

HttpSecurity是一个immutable类，用于设置http请求拦截规则

AuthenticationManagerBuilder

用于构建AuthenticationManager，可以构建出基于特定环境和边界地构建AuthenticationManager的功能
