OAuth2是什么？

  • OAuth2.0是OAuth1.0（Open Authorization，开放授权）协议的延续版本，但不向后兼容OAuth1.0，而OAuth是一个受权的开放标准，即公开的标准，是一个授权协议;
  • 在LETF-rfc6749中的标题为The OAuth 2.0 Authorization Framework，可以看出OAuth2.0是一个授权框架并且能够完成“安全委托访问”，通俗点说就是你可以授权b网站去访问你存储在a网站上的一些信息，而不会泄露除了这个“一些信息”之外的内容；
  • Oauth2的四种授权类型
    • 授权码模式
    • 简单模式
    • 密码模式
    • 客户端模式

网上关于Oauth2协议介绍的文章有很多，这里不做过多的叙述，深入了解推荐以下文章和书籍：

  • 阮一峰的《理解OAuth 2.0》,直接百度名称就有了，这文章是对LETF-rfc6749的白话简述
  • 书籍《OAuth2实战》对OAuth2作出了详细的介绍

关于jwt的那些事

​ jwt的出现无疑是为了解决session占用服务器内存和节点间session需要同步的问题，因为日益增长的用户、成熟的技术和应用架构对于session而言已经慢慢成为了累赘; 如在微服务架构中就提倡节点无状态，其实很容易理解，因为对于灵活的节点扩展而言，如果节点存在状态的话那么在拓展节点之后就会可能会出现问题，所以本地缓存、session等让服务器节点产生状态的操作都是尽量避免的;

​ 如果使用jwt的话，就意味着服务器并不对会话信息进行维护而是让客户端来维护，这是一个职责的转移，所以jwt的相关内容应该是尽量避免持久化的，服务器的职责只是对jwt进行验证，至于jwt是谁发过来的不管，只要jwt合法服务器就理所当然接受请求处理请求，所以如何保证jwt的使用安全是需要客户端合理设计的，如可以使用https协议对通信进行加密处理;为什么突然提到这个是因为看到网上一些文章在讨论这问题，这是我对与jwt的理解，所以如果使用jwt如果不是没办法的话，最合理的做法就是服务器只对jwt做验证操作;

​ 还需要注意的是，jwt只是token的一种，jwt是不应该持久化的，而bearer令牌需要持久化，因为bearer不像jwt那样能够针对结构化内容校验，bearer只是一串字符;

  1. 将”头部”和“负载”decode，返回一个DecodeJWT
  2. 验证”头部”的加密算法是否和服务器一直，不一直接报错

使用JDK内置的证书管理工具keytool实现token的RSA非对称加密

推荐文章：https://blog.csdn.net/liumiaocn/article/details/61921014

1）切换到工程的resources目录执行一下命令生产对应keystore文件

keytool -genkey -alias oauth2 -keyalg RSA -keystore oauth2.keystore -keysize 2048
  • genkey：别名
  • keyalg：密钥使用的算法
  • keystore：文件的名称
  • keysize：密钥的长度

2）执行上面的生成命令后，会提建议执行一下命令

keytool -importkeystore -srckeystore oauth2.keystore -destkeystore oauth2.keystore -deststoretype pkcs12

3）打印出keystore中的私钥和公钥

keytool -list -rfc --keystore oauth2.keystore | openssl x509 -inform pem -pubkey

授权服务器配置

CustomTokenEnhancer：该类用于扩展token内容，默认情况下spring security oauth2返回的jwt之包含用户信息的用户名称，如果需要添加内容到jwt中，就需要扩展

@Component
public class CustomTokenEnhancer implements TokenEnhancer {

    @Override
    public OAuth2AccessToken enhance(OAuth2AccessToken accessToken, OAuth2Authentication authentication) {

        DefaultOAuth2AccessToken customAccessToken = (DefaultOAuth2AccessToken) accessToken;
        // 授权服务器默认返回token类型是bearer
        // 因为当前案例使用的token是jwt而不是默认的bearer，所以最好修改一下对应起来，不修改也可以只是看起来很奇怪，显示的是bearer但用的jwt，但为了更加明确建议修改
        customAccessToken.setTokenType("jwt"); 

        // 设置自定义字段
        final Map<String, Object> additionalInfo = new HashMap<>(8);
        // User user = (User) authentication.getUserAuthentication().getPrincipal();
        additionalInfo.put("user_cardid", "441802199682648829");
        customAccessToken.setAdditionalInformation(additionalInfo);

        return customAccessToken;
    }
}

Oauth2AuthorizationServerConfig：授权服务器的主要配置

@Configuration
@EnableAuthorizationServer
public class Oauth2AuthorizationServerConfig extends AuthorizationServerConfigurerAdapter {

    @Autowired
    private PasswordEncoder passwordEncoder;

    // 刷新令牌使用
    @Autowired
    @Qualifier("userDetailsServiceBean")
    private UserDetailsService userDetailsServiceBean;

    @Autowired
    @Qualifier("authenticationManagerBean")
    private AuthenticationManager authenticationManager;


    @Bean
    public JwtAccessTokenConverter accessTokenConverter() {
        JwtAccessTokenConverter jwtAccessTokenConverter = new JwtAccessTokenConverter();

        // 对称加密
        //jwtAccessTokenConverter.setSigningKey("secret"); // secret为密钥

        // 非对称加密
        // 非对称加密，校验只需要使用公钥即可, 如jwt.io校验
        final String privateKey = "123456";
        final String alias = "oauth2";
        final String filePath = "oauth2.keystore";
        KeyStoreKeyFactory keyStoreKeyFactory =
                new KeyStoreKeyFactory(new ClassPathResource(filePath), privateKey.toCharArray());
        jwtAccessTokenConverter.setKeyPair(keyStoreKeyFactory.getKeyPair(alias));

        return jwtAccessTokenConverter;
    }

    // 使用jwt作为token
    @Bean
    public TokenStore tokenStore() {
        // 需要一个信息转换器
        return new JwtTokenStore(accessTokenConverter());
    }

    @Override
    public void configure(AuthorizationServerSecurityConfigurer security) throws Exception {
        // 设置oauth2端点访问权限
        security.tokenKeyAccess("permitAll()") // 默认为denyAll(),拒绝访问
                .checkTokenAccess("isAuthenticated()") // 默认为denyAll(),拒绝访问
                .allowFormAuthenticationForClients(); // 支持获取令牌无需登录
    }


    // 注入自定义的token扩展
    @Autowired
    private CustomTokenEnhancer customTokenEnhancer;

    // 1. 配置oauth2授权服务器端点
    @Override
    public void configure(AuthorizationServerEndpointsConfigurer endpoints) throws Exception {

        // 将自定义的TokenEnhancer添加到链上，否则会覆盖
        TokenEnhancerChain tokenEnhancerChain = new TokenEnhancerChain();
        tokenEnhancerChain.setTokenEnhancers(Arrays.asList(customTokenEnhancer, accessTokenConverter()));

        endpoints
                .authenticationManager(authenticationManager) // 认证管理器对接security
                .userDetailsService(userDetailsServiceBean) // 刷新令牌需要使用其校验用户
                .tokenStore(tokenStore()) // 设置token store
                // jwt配置
                .tokenEnhancer(tokenEnhancerChain); // 添加token扩展，如果不对token扩展使用下面方式直接配置
//                .accessTokenConverter(accessTokenConverter());
    }

    // 2. 配置基于内存的客户信息
    // 说明：客户端如果需要使用授权系统来获取权限，那么就需要在授权系统上注册自己的信息，这个上面已经提到过
    // 实际开发当中，对于客户端的注册信息，授权服务器应该提供相应的api接口来提供给其他客户端应用进行动态注册并将注册信息持久化，而不是写死
    @Override
    public void configure(ClientDetailsServiceConfigurer clients) throws Exception {
        clients.inMemory()
                .withClient("client-app") // 客户端id
                .redirectUris("https://www.baidu.com") // 客户端的回调地址（既授权码返回给谁），这里我随便写了一个
                // 如果secret不使用passwordEncoder编码，会出现"Bad client credentials"
                .secret(passwordEncoder.encode("secret")) // 客户端密钥
                .accessTokenValiditySeconds(1800) // token有效时间
                .refreshTokenValiditySeconds(3600) // 刷新token有效时间，一般需要比token大，具体大多少根据业务需要
                // 当前授权服务器支持的授权模式
                .authorizedGrantTypes("refresh_token", "authorization_code")
                .scopes("READ_USER_INFO", "READ_PROJECT_INFO"); // 支持的授权域（既允许客户端的操作范围;根据需要来设置，不是客户端注册的部分）
    }
}

@RestController
@RequestMapping("/resource")
public class UserController {

    // 需要拥有 USER 角色才能够访问
    @PreAuthorize("hasRole('USER')")
    @GetMapping("/user/{id}")
    public ResponseEntity<UserInfo> userInfo(@PathVariable Integer id) {

        UserInfo user = UserInfo.builder()
                .id(id)
                .name("zhangsan")
                .build();

        return new ResponseEntity(user, HttpStatus.OK);
    }

    // 需要token的授权区域（范围）在 READ_USER_INFO 才能够访问
    @PreAuthorize("#oauth2.hasScope('READ_USER_INFO')")
    @GetMapping("/user2/{id}")
    public ResponseEntity<UserInfo> userInfo2(@PathVariable Integer id) {

        UserInfo user = UserInfo.builder()
                .id(id)
                .name("zhangsan")
                .build();

        return new ResponseEntity(user, HttpStatus.OK);

http://localhost:8888/oauth/authorize?client_id=client-app&response_type=code&scope=READ_USER_INFO

注意：这里的scope不是必须的，如果不指定获取的权限范围的话，则默认获取全部范围;

注意和拓展

  • 对于授权码模式中的code，如果自定义的话最好设置较短的超时，例如微信开发平台上的openid貌似就10秒超时。对于上面这个案例而言 spring security oauth2 返回的code是一次性的,即只能够用一次，用完就消失了。以下为RandomValueAuthorizationCodeServices源码：

    public OAuth2Authentication consumeAuthorizationCode(String code)
    throws InvalidGrantException {
    // 在获取授权信息的同时删除code
    OAuth2Authentication auth = this.remove(code);
    if (auth == null) {
        throw new InvalidGrantException("Invalid authorization code: " + code);
    }
    return auth;
}

  • 如果授权码模式不想手动点击确认授权，可以在授权服务器的client配置处将AutoApprove设置为true

  • 验证token：如果使用jwt的话，除了使用相关api可以验证之外，还可以通过网站jwt.io来自行观察一下

  • state参数：对于客户端应用而言，一接受到回调地址中的code就会将其用于请求授权服务器，但有没有想过这个code并不是授权服务器回调回来的而是可能有人别有用心恶意调用客户端的回调地址，为了解决这种恶意的调用，可以在请求授权服务器时带上一个state=任意标记参数，授权服务器接收到该参数后会连同code一并返回，客户端只需要判断一下state是否和发出请求时一致既可避免恶意的调用和csrf攻击（跨站访问伪造）

了解Spring Security OAuth2

关于自动配置

经过上面对ouath2的简单使用过后，不难发现，对于授权服务器和资源服务器的基本原理和使用是差不多的;

从用例中引出如下问题：

  1. Spring Security OAuth2无疑是基于SpringSecurity的，那么它是如何建立关系的
  2. 如何维护token：TokenStore和TokenConverter
  3. 如何维护客户端注册信息（详细信息）：ClientDetailsService
  4. 如何处理oauth2端点请求

带着以上的疑问进入spring-security-oauth2-autoconfigure包的自动配置类OAuth2AutoConfiguration，以下为主要部分：

// ...
// 可以自动配置的有如下内容
@Import({ OAuth2AuthorizationServerConfiguration.class,
        OAuth2MethodSecurityConfiguration.class, OAuth2ResourceServerConfiguration.class,
        OAuth2RestOperationsConfiguration.class })
@AutoConfigureBefore(WebMvcAutoConfiguration.class)
// ...
public class OAuth2AutoConfiguration {
    // ...
}

OAuth2AuthorizationServerConfiguration：授权服务器

OAuth2ResourceServerConfiguration：资源服务器

OAuth2MethodSecurityConfiguration：对授保护方法增强处理，如Oauth2表达式处理

OAuth2RestOperationsConfiguration：根据配置文件进行client相关的条件配置

授权服务器自动配置

@Configuration
// 当使用@EnableAuthorizationServer时启动自动配置
// EnableAuthorizationServer注解触发以下两个配置
//     AuthorizationServerEndpointsConfiguration.class // Oauth2端点配置
//     AuthorizationServerSecurityConfiguration.class // SpringSecurity配置
@ConditionalOnClass(EnableAuthorizationServer.class)
@ConditionalOnMissingBean(AuthorizationServerConfigurer.class) // 必须没有自定义配置
@ConditionalOnBean(AuthorizationServerEndpointsConfiguration.class) // 上下文存在该bean实例
@EnableConfigurationProperties(AuthorizationServerProperties.class) // 应用授权服务器配置参数
@Import(AuthorizationServerTokenServicesConfiguration.class) // 根据配置文件token参数条件进行条件配置
public class OAuth2AuthorizationServerConfiguration
        extends AuthorizationServerConfigurerAdapter {

    // 客户注册信息，生成逻辑参考BaseClientDetailsConfiguration
    private final BaseClientDetails details;
    // SpringSecurity的认证管理器
    private final AuthenticationManager authenticationManager;
    // token操作类
    private final TokenStore tokenStore;
    // token维护类
    private final AccessTokenConverter tokenConverter;
    // 授权服务器配置参数bean
    private final AuthorizationServerProperties properties;

    // 自动注入参数
    public OAuth2AuthorizationServerConfiguration(BaseClientDetails details,
            AuthenticationConfiguration authenticationConfiguration,
            ObjectProvider<TokenStore> tokenStore,
            ObjectProvider<AccessTokenConverter> tokenConverter,
            AuthorizationServerProperties properties) throws Exception {
        //...
    }

    // 客户端注册信息配置
    @Override
    public void configure(ClientDetailsServiceConfigurer clients) throws Exception {

        ClientDetailsServiceBuilder<InMemoryClientDetailsServiceBuilder>
            .ClientBuilder builder = clients
            // 基于内存生产客户端注册信息
            // 参考BaseClientDetailsConfiguration
            .inMemory()
            .withClient(this.details.getClientId()); // uuid
        builder.secret(this.details.getClientSecret()) // uuid
                .resourceIds(this.details.getResourceIds().toArray(new String[0]))
                .authorizedGrantTypes(
                        this.details.getAuthorizedGrantTypes().toArray(new String[0]))
                .authorities(
                        AuthorityUtils.authorityListToSet(this.details.getAuthorities())
                                .toArray(new String[0]))
                .scopes(this.details.getScope().toArray(new String[0]));

        if (this.details.getAutoApproveScopes() != null) {
            builder.autoApprove(
                    this.details.getAutoApproveScopes().toArray(new String[0]));
        }
        if (this.details.getAccessTokenValiditySeconds() != null) {
            builder.accessTokenValiditySeconds(
                    this.details.getAccessTokenValiditySeconds());
        }
        if (this.details.getRefreshTokenValiditySeconds() != null) {
            builder.refreshTokenValiditySeconds(
                    this.details.getRefreshTokenValiditySeconds());
        }
        if (this.details.getRegisteredRedirectUri() != null) {
            builder.redirectUris(
                    this.details.getRegisteredRedirectUri().toArray(new String[0]));
        }
    }

    // 配置oauth2端点
    // 参考自动配置类AuthorizationServerEndpointsConfiguration
    @Override
    public void configure(AuthorizationServerEndpointsConfigurer endpoints)
            throws Exception {
        // tokenStore使用的内存生产，以下两个都是null
        if (this.tokenConverter != null) {
            endpoints.accessTokenConverter(this.tokenConverter);
        }
        if (this.tokenStore != null) { 
            endpoints.tokenStore(this.tokenStore);
        }

        // 注意：清楚看到当授权服务器支持密码授权模式的话就需要用到security的AuthenticationManager
        if (this.details.getAuthorizedGrantTypes().contains("password")) {
            endpoints.authenticationManager(this.authenticationManager);
        }
    }

    // 客户端注册信息配置（绑定到授权服务器）
    @Override
    public void configure(AuthorizationServerSecurityConfigurer security)
            throws Exception {
        // 获取SpringSecurity中的密码编码器
        security.passwordEncoder(NoOpPasswordEncoder.getInstance());

        // 参考AuthorizationServerSecurityConfigurer
          // 如果上面端点配置和配置文件中没有配置则为空，但默认该端点拦截级别为"denyAll()"
        if (this.properties.getCheckTokenAccess() != null) {
            security.checkTokenAccess(this.properties.getCheckTokenAccess());
        }
        // 如果上面端点配置和配置文件中没有配置则为空，但默认该端点拦截级别为"denyAll()"
        if (this.properties.getTokenKeyAccess() != null) {
            security.tokenKeyAccess(this.properties.getTokenKeyAccess());
        }
        // 如果上面端点配置和配置文件中没有配置则为空
        if (this.properties.getRealm() != null) {
            security.realm(this.properties.getRealm());
        }
    }

    // 以INFO日志打印自动生产的客户端注册信息
    // 例子如下：
    // security.oauth2.client.client-id = 2f943783-66eb-4f27-b8ad-5b19c422891f
    // security.oauth2.client.client-secret = 20d598a7-397c-475a-9991-ea9e57e5187c
    @Configuration
    protected static class ClientDetailsLogger implements InitializingBean {

        private final OAuth2ClientProperties credentials;

        protected ClientDetailsLogger(OAuth2ClientProperties credentials) {
            this.credentials = credentials;
        }

        public void afterPropertiesSet() {
            init();
        }

        public void init() {
            String prefix = "security.oauth2.client";
            boolean defaultSecret = this.credentials.isDefaultSecret();
            logger.info(String.format(
                    "Initialized OAuth2 Client%n%n%s.client-id = %s%n"
                            + "%s.client-secret = %s%n%n",
                    prefix, this.credentials.getClientId(), prefix,
                    defaultSecret ? this.credentials.getClientSecret() : "****"));
        }

    }

    // 自动生产的客户端注册信息
    @Configuration
    @ConditionalOnMissingBean(BaseClientDetails.class)
    protected static class BaseClientDetailsConfiguration {

        private final OAuth2ClientProperties client;

        protected BaseClientDetailsConfiguration(OAuth2ClientProperties client) {
            this.client = client;
        }

        @Bean
        @ConfigurationProperties(prefix = "security.oauth2.client")
        public BaseClientDetails oauth2ClientDetails() {
            BaseClientDetails details = new BaseClientDetails();
            if (this.client.getClientId() == null) {
                // uuid
                this.client.setClientId(UUID.randomUUID().toString());
            }
            details.setClientId(this.client.getClientId());
            // this.client.getClientSecret() 也是返回uuid
            details.setClientSecret(this.client.getClientSecret());
            // 支持所有授权模式
            details.setAuthorizedGrantTypes(Arrays.asList("authorization_code",
                    "password", "client_credentials", "implicit", "refresh_token"));
            // 默认授权为USER
            details.setAuthorities(
                    AuthorityUtils.commaSeparatedStringToAuthorityList("ROLE_USER"));
            // 回调地址
            details.setRegisteredRedirectUri(Collections.<String>emptySet());
            return details;
        }

    }
}
