关于BeanFactory和ApplicationContext的区别

  • BeanFactory的主要功能

    • 依赖查找
    • 依赖注入
    • Bean管理（注册、注销）

  • Application的主要功能

    • 资源管理（ResourceLoader）
    • 事件传播（ApplicationEventPublisher）
    • 国际化（MessageSource）

  • 之间的关系如下，可以看到它们之间是组合的关系。Application是BeanFactory的扩展

refresh() - 刷新启动容器

该方法继承自AbstractApplicationContext类

解析出配置类

invokeBeanFactoryPostProcessors -> invokeBeanFactoryPostProcessors -> invokeBeanDefinitionRegistryPostProcessors(ConfigurationClassPostProcessor) -> postProcessBeanDefinitionRegistry

finishBeanFactoryInitialization() - 实例化Bean

beanFactory.preInstantiateSingletons()

AbstractBeanFactory#doGetBean

createBean() 实际实现调用 AbstractAutowireCapableBeanFactory#doCreateBean 来创建

createBeanInstance() 内调用的是 AbstractAutowireCapableBeanFactory#createBeanInstance

instantiateUsingFactoryMethod() 内调用的是 ConstructorResolver#instantiate

  • 可以看到 Singleton Bean 实例化逻辑位于 beanFactory.getInstantiationStrategy().instantiate()
  • 目前 Spring 中实例化 Bean 的策略主有两种：
    • SimpleInstantiationStrategy：采用反射进行实例化
    • CglibSubclassingInstantiationStrategy（默认）：通过 CGLIB 生成子类实例化（继承自SimpleInstantiationStrategy），即这时返回的是一个代理

关于InstantiationStrategy：InstantiationStrategy接口

无参构造器方式 - SimpleInstantiationStrategy#instantiate

有参构造器方式 - SimpleInstantiationStrategy#instantiate

工厂方法方式 - SimpleInstantiationStrategy#instantiate
